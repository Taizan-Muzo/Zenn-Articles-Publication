# はじめに

LLMの強化学習（RL）が推論能力を引き出す手段として定着しつつあるいま、ひとつの盲点が指摘されている。RLによる学習の効率を高める研究は、報酬設計やデータ合成に集中してきたが、**モデルの内部表現そのものが持つ構造**を活用するアプローチはほとんどなかった。

LongAct（arXiv:2604.14922）は、この空白を埋める研究だ。北京大学と上海交通大学のチームが提案したこの手法は、長文脈処理時のQuery/Keyベクトルに現れる**大振幅アクティベーション**に着目し、その重要性を認識した上でRL訓練を**スパースな重み更新**に切り替える。

驚くべきことに、この変更だけでLongBench v2で+8%、しかも**推論時のオーバーヘッドはゼロ**。理論も実装も驚くほどシンプルだが、実験結果は目を見張るものがある。

# TL;DR

- 長文脈を処理する際、Q/Kベクトルの特定次元に大振幅アクティベーションが集中する現象を発見
- この大振幅次元の重みだけを更新するスパース勾配マスク「LongAct」を提案
- DAPO/GRPOを含む複数RLアルゴリズムで一貫して性能向上（LongBench v2 36.73、+8%）
- 推論時はマスクを破棄するだけなので、遅延ゼロ・パラメータ数も変更なし
- 大振幅アクティベーションを除去するとモデルが即座に崩壊するという強い因果的証拠

# なぜこの問題が重要なのか

最近のLLMのRL研究は、DeepSeek-R1以降「推論能力を引き出す」方向に加速している。しかし、長文脈（数万〜数十万トークン）でのRL訓練には特有の難しさがある。

第一に、**計算コスト**が膨大になる。シーケンス長が32K〜128Kになれば、attentionの計算量は二次的に増大し、勾配計算のメモリも逼迫する。

第二に、**勾配の希薄性**という問題がある。長文脈の推理タスクでは、入力全体のうち実際に回答生成に寄与するのは一部のトークンに過ぎない。それにもかかわらず、従来のRL訓練は全パラメータに均等に勾配を流すため、非関連情報のノイズによって有用なシグナルが希釈されてしまう。

LongActは、モデル自身が「どこが重要か」をアクティベーションの振幅という形で知っていることに着目した。この直感は、モデル量子化の分野で古くから知られていた知見——**大振幅の重みがモデルの性能に決定的な役割を果たす**——に根ざしている。

# LongActの中身

## 発見：Q/Kベクトルの大振幅アクティベーション

論文が最初に示したのは、長い入力を処理する際、Queryベクトル$Q$とKeyベクトル$K$の特定の次元に継続的に大きな値が現れるという観察だ。著者らは、バッチ次元で平均化した$\ell_2$ノルムによって「振幅行列」$M^Q_{h,d}$を定義した。

$$M^Q_{h,d} = \frac{1}{B}\sum_{i=1}^{B}\sqrt{\sum_{s=1}^{S}(Q^{(i)}_{s,h,d})^{2}}$$

この行列を可視化すると、各ヘッドで少数の次元だけが突出して大きい値を持つことがわかる。つまり、**長文脈の処理は本質的にスパースな構造**を持っているのだ。

![均一更新 vs スパース更新の概念図](/images/longact-sparse-qk-rl/fig1_uniform_vs_sparse.png)
*従来のRLは全重みを均等に更新するが、LongActは大振幅次元（赤い箇所）だけを選択的に更新する*

## スパース勾配マスクの仕組み

LongActの仕組みは驚くほどシンプルだ。以下の3ステップで構成される：

**ステップ1：ヘッド内Top-k選択**

グローバルなtop-kではなく、各attentionヘッドごとに上位$k$個の次元を選ぶ。これによりマルチヘッド構造が保持される。

$$\mathcal{K}_h = \left\{d \mid d \in \mathop{\arg\max}_k \{M^Q_{h,d'}\}_{d'=0}^{D-1}\right\}$$

**ステップ2：二値勾配マスクの生成**

選ばれた次元に対応する重み行だけを更新対象とする二値マスク$\mathbf{G}^Q$を生成する。

**ステップ3：逆伝播でのマスク適用**

$$\nabla W_Q \leftarrow \nabla W_Q \odot \mathbf{G}^Q, \quad \nabla W_K \leftarrow \nabla W_K \odot \mathbf{G}^K$$

ここで重要なのは、**スパース更新は$W_Q$と$W_K$にのみ適用**され、$W_V, W_O, MLP$層は通常通りフル更新される点だ。推論時にはマスクを破棄するため、モデルは標準の密なTransformerとして動作する——追加の遅延もパラメータ増もない。

デフォルトのスパース率$\lambda$は**0.3**（上位30%のみ更新）。消融実験でこの値が最適であることが確認されている。

## 大振幅アクティベーションの因果的証拠

もっとも説得力のある実験は「摂動分析」だ。LongBench v2の実際のテストケース503問に対して、Qwen3-8BのQ/Kアクティベーションを意図的に操作した結果が以下の通り：

| 条件 | 正解数 (503問中) |
|:---|:---:|
| 元のモデル | 173 / 503 |
| 大振幅以外を除去 | 108 / 503（低下するが崩壊はしない） |
| **大振幅を除去** | **0 / 503（出力が「33333...」に退化）** |

大振幅アクティベーションを除去するとモデルが**即座に崩壊**するという事実は、これらの要素が長文脈推理にとって不可欠であることを強く示唆している。

# 実験結果

## LongBench v2

![LongBench v2での比較](/images/longact-sparse-qk-rl/fig2_longbench_comparison.png)
*Qwen3-8Bで36.73（DAPO単独から+3.93）、Qwen3-4Bで34.24（同+3.82）*

LongBench v2は、簡単・困難・短・中・長の5つの軸で評価される長文脈推論ベンチマークだ。LongAct + DAPOの組み合わせは、SFT単独に対して約+10ポイント、DAPO単独に対しても+3.8〜+3.9ポイントの改善を達成した。

とくに「困難」カテゴリでの伸びが著しい。Qwen3-8Bでは困難問題の正解率が28.30%から35.93%へと+7.63ポイント——**相対で27%の改善**だ。

## RULER-128K

![RULER-128Kのタスク別詳細](/images/longact-sparse-qk-rl/fig3_ruler_breakdown.png)
*Retrieval（QA）タスクで顕著な改善。4Bモデルで+3.4ポイント*

RULERベンチマークは128Kコンテキストでの情報検索能力を評価する。LongActはNIAH（Needle In A Haystack）タスクで一貫した向上を見せ、とくにRetrieval（QA）タスクでDAPOに対して有意な改善を記録した。

## 短文脈への悪影響はない

| モデル | GSM8K | HumanEval | TruthfulQA |
|:---|:---:|:---:|:---:|
| SFT | 71.94 | 68.90 | 67.76 |
| SFT + DAPO | 78.08 | 69.51 | 68.95 |
| **SFT + LongAct** | **80.13** | **73.17** | **69.52** |

ここが意外だった。長文脈最適化手法は往々にして短文脈性能を犠牲にするが、LongActは**短文脈ベンチマークでもDAPO単独を上回る**。GSM8Kで+2.05、HumanEvalで+3.66——長文脈特化のはずなのに、一般推論能力まで向上したのだ。

## RLアルゴリズムとの互換性

![消融研究：スパース率とアルゴリズム互換性](/images/longact-sparse-qk-rl/fig4_ablation_study.png)
*左：スパース率30%が最適。右：DAPO/GRPO/CLIP-conv/KL-convの全てで改善*

LongActは特定のRLアルゴリズムに依存しない。DAPOだけでなく、GRPO、CLIP-constrained、KL-constrainedのいずれでも一貫して性能向上を示した。DAPOとの組み合わせが最も高いスコアを記録したが、GRPOでも35.04（8B）とDAPO単独（32.80）を上回っている。

# 考察

## シンプルさが強み

LongActの最大の魅力は、その実装の容易さにある。勾配に二値マスクを掛けるだけなので、既存のRL訓練パイプライン（LLaMA-FactoryやveRLなど）に数行で組み込める。新しい損失関数やアーキテクチャ変更は不要で、推論時のオーバーヘッドもゼロ——実用性の観点からほぼ完璧に近い。

## 「量子化の知見」をRLに持ち込んだ発想の転換

モデル量子化の分野では「大振幅の重み＝重要な情報」という認識が定着している（AWQなど）。LongActの面白さは、この知見を**訓練時の勾配選択**に転用した点にある。更新の対象を絞ることで、非関連な勾配による「最適化のノイズ」を減らし、本当に重要な重みにリソースを集中させている。

## スパース率30%という「スイートスポット」

消融実験から、30%という数字は偶然ではないことがわかる。20%では情報が不足し、40%ではノイズが混ざり始める。この境界はタスクやモデルサイズに応じて適応的に決定できる可能性があり、今後の発展が期待される。

## 残された課題

- 大振幅アクティベーションが**なぜ**長文脈処理で現れるのか、理論的な裏付けがまだ薄い
- スパース率のタスク依存性の分析が不足している（難易度や文脈長によって最適値が変わる可能性）
- $W_Q, W_K$以外のレイヤー（MLPなど）でも同様の現象が起きるかの検証が未着手

# まとめ

LongActは「RLの勾配をどこに流すか」という極めて根本的な問いに対し、モデル内部のアクティベーションという客観的な証拠に基づく答えを提示した。大振幅Q/Kアクティベーションだけを更新対象にするというアイデアは、実装が数行で済むにもかかわらず、LongBench v2で+8%という実質的な改善をもたらす。

もっとも印象的なのは、この手法が**推論コストを一切増やさない**ことだ。マスクは訓練時だけ使われ、推論時には完全に透明になる。これは実用上非常に大きな利点で、既存のRLパイプラインへの統合も容易だ。

長文脈LLMの発展において、「どのように報酬を設計するか」「どのようなデータで訓練するか」に次ぐ第三の軸——**「どのパラメータを更新するか」**——がこれから重要になっていくとすれば、LongActはそのパイオニアとしての位置を占めることになるだろう。
