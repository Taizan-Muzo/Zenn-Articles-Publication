## TL;DR

- **RLVR**（GRPOやDAPO）はLLMの推論能力を 크게伸ばすが、同時に深刻な**校準退化（Calibration Degeneration）**——錯誤答案にたいする過度の自信——を引き起こす
- 既存の対策（校準目的を損失関数に合成）は効果が薄い。論文はここに**根本的な勾配衝突**が存在することを数学的に証明した
- **DCPO**（Decoupled Calibration Policy Optimization）は**構造・報酬・勾配の三重解耦**でこれを回避。GRPOと同等の精度を維持しながら校準誤差を最低にする
- 追加パラメータなし・追加ネットワークなし・計算オーバーヘッドほぼゼロで実装できる

---

## 背景：RLVRが抱える「自信過剰」問題

GRPOやDAPOといった**RLVR（Reinforcement Learning from Verifiable Rewards）** 系の手法は、検証可能な外部奖励（正誤判定）を сигнал に使うことで、LLMの数学的推論能力を飛躍的に伸ばした。

しかし、そこに一抹の影がある。RLVRで訓練されたモデルは、**錯誤答案に対して異常に高い置信度を表示する**ようになる。

言い換えると、「全然わかっていないのに自信満々」という困った状態が起きる。モデルの出す確率が信頼できないと、デプロイ時にどれを信用すればいいか判断できない。

### 既存の対策とその限界

多くの研究がこの問題に取り組んできた。校準目的（Calibration Loss）をRL目的と jointly に最適化するアプローチが代表的だ。

$$
L = L_{\text{RL}} + \lambda \cdot L_{\text{cal}}
$$

しかし実際の効果は限定的だった。DCPOの著者がこの理由を理論的に解明している。

---

## 核心発見：最大化と最小化は相反する

論文の最も重要な貢献は、**「精度向上」と「校準誤差最小化」の間で勾配衝突が発生すること**を証明した点にある。

### 勾配衝突の直感的な説明

ある困難な問題を考えてほしい。モデルの実際の正答率が40%しかないのに、置信度が90%を示しているケース。在这种状态下会发生什么？

| 最適化目標 | 勾配の方向 | 具体的な要求 |
|-----------|-----------|------------|
| **RL（精度最大化）** | 上昇 | 「この答案是正しかった」という信号 → ロジットを引き上げる |
| **Calibration（校準）** | 下降 | 「自信过头了（過信）」という信号 → ロジットを引き下げる |

两つの勾配が**180度反対方向**を向いている。结果、訓練が不安定になるか、あるいはモデルの推論能力そのものが低下してしまう。

### 跷跷板效应（シーソーの問題）

既存の jointly 最適化では、こうなる：

- $\lambda$ を小さくすると：精度は高いが依然として過信が解消されない
- $\lambda$ を大きくすると：置信度は下がるが、正解の確信まで下がって精度も落ちる

**精度と校準を同時に追求しようとする限り、このトレードオフは避けられない。**

![勾配衝突の概念図：180度反対方向の勾配がシーソーの両側を揺らす](/images/dcpo-decoupling-calibration/fig1_gradient_conflict.png)
*RL勾配とCal勾配が同じロジットに対して反対方向に作用するため、訓練が不安定化。正解は「シーソーを разделить」して別々に扱うこと。*

---

## DCPO：三重解耦による解決策

勾配衝突を避ける唯一のアプローチは、**推論目標と校準目標を完全に分离する**こと。DCPOはこれを三つのレベルで実行する。

### 1. 構造の解耦（Structure Decoupling）

従来のRLVR出力：

```
[問題] → [推理ステップ] → [最終回答]
```

DCPOの出力には**専用の `<confidence>` トークンが末尾に追加**される：

```
[問題] → [推理ステップ] → [最終回答] → <confidence> 0.85 </confidence>
```

これで「推理能力强いトークン」と「校準を担当するトークン」が**時系列的に分离**される。結構な改善だ。

### 2. 報酬の解耦（Reward Decoupling）

| 報酬の種類 | 対象トークン | 計算方法 |
|-----------|-----------|---------|
| **推理報酬** $r_a$ | 推理ステップ + 最終回答 | 判題器verified → 正解=1 / 不正解=0 |
| **校準報酬** $r_c$ | `<confidence>` トークンのみ | 対数损失の負値 |

推理と校準が**別のトークン序列**で独立に计算される。これにより互いの干涉が構造的に防止される。

### 3. 勾配の解耦（Gradient Decoupling）

これが最も重要な設計。

```python
# 疑似コード
token_level_advantages = (reason_mask * A_acc) + (conf_mask * A_cal)
policy_loss = -(log_probs * token_level_advantages).mean()
```

Advantage Masking により：

- **推理トークン** には推理の优势 $A_{\text{acc}}$ 만 적용
- **校準トークン** には校準の优势 $A_{\text{cal}}$ 만 적용

校準目的函数的勾配が推理トークンの勾配に影響を与えることを、物理的に防止する。

![三重解耦の概観](/images/dcpo-decoupling-calibration/fig2_triple_decoupling.png)
*左から順に：構造解耦（専用トークン）、報酬解耦（独立奖励信号）、勾配解耦（Advantage Masking）。*

## 実験結果

### 方法間比較

![精度と校準のトレードオフ比較](/images/dcpo-decoupling-calibration/fig3_methods_comparison.png)
*GRPO，联合优化，DCPOの3手法を比較。DCPOだけが精度を保ちながら最高の校準性能を達成。*

| 手法 | 精度 | ECE（↓） | 備考 |
|------|------|---------|------|
| GRPOのみ | 76% | 0.18 | 推論能力强だが過信 |
| 联合最適化（RL+Cal） | 60% | 0.10 | 勾配衝突で推理能力が低下 |
| **DCPO** | **76%** | **<0.05** | 両方满足 |

DCPOはGRPOと同等の精度を維持しながら、校準誤差を**0.18から0.05未満**に改善した。

### 置信度分布の変化

![GRPOとDCPOの置信度分布比較](/images/dcpo-decoupling-calibration/fig4_confidence_distribution.png)
*GRPO：错误答案の置信度が0.5以上に集中（過信）。DCPO：错误答案の置信度分布がより低い值に移動し、校準良好的。*

### 実装コスト

| 項目 | DCPO |
|------|------|
| 追加パラメータ | ❌ なし |
| 追加ネットワーク | ❌ なし |
| 追加フォワードパス | ❌ なし |
| 訓練オーバーヘッド | **ほぼゼロ** |

三重解耦はすべて既存インフラの上で動作し、新たなネットワークや追加計算を必要としない。

## 考察：この論文が示唆するもの

DCPOの技術的貢献を超えて注目すべきは、**「 jointly 最適化 всегда 正しいわけではない」**という方法論的な警鐘だ。

機械学習では往々にして、多个目的を 하나의損失関数に合成して同時に最適化しようとする。しかし、各目的間の勾配方向が冲突している場合、简单地相加はどちらも 달성不了 results。

この知見はRLVR以外にも适用できる。 например：

- **報酬整形と正則化**の冲突
- **正確性と公平性**の冲突
- **高速推理と高精度**の冲突

複数目的の最適化に直面した時、「解耦して别々に处理する」という选择的は、强有力的归纳バイアスになる。

## まとめ

DCPOは、RLVR訓練における校準退化问题的根本原因を数学的に解明し、**勾配衝突を避ける唯一の方法は目的を分离すること**であることを示した。

- **理論**：精度最大化と校準誤差的最小化は180度反対の勾配を持つ
- **解決策**：三重解耦（構造・報酬・勾配）で推論と校準を完全に分離
- **効果**：GRPO比で同精度 + ECE 0.18 → 0.05未満
- **実装**：追加パラメータ不要、オーバーヘッドほぼゼロ

RLVR系手法の普及に伴い、推論能力と信頼性の両立はますます重要になる。DCPOはそこに一つのエレガントな答えを提示した论文だ。
