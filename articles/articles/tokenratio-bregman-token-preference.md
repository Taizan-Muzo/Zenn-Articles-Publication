## TL;DR

- **TBPO**（Token-level Bregman Preference Optimization）は、DPOを**トークン単位**に拡張し、**Bregman divergenge**を用いた密度比マッチングで最適なポリシーを直接回復する新フレームワーク
- **TBPO-Q**はQ関数をスコアに用いる変種で、軽量MLPベースラインヘッドを追加（stop-gradientでLM本体は不変）
- **TBPO-A**はアドバンテージ関数をスコアに用いる変種で、**追加パラメータゼロ**で実装
- **SBA**（Scaled Basu's power divergence）という凸関数で統一的に定式化し、ロジスティック回帰・KLIEP・LSIFを特殊ケースとして含む
- Llama 3 8Bで平均スコア**70.50**（DPO 67.31を上回る）、かつ応答長は**最短**

## なぜこの論文が重要か

LLMのアライメント手法として、RLHFに代わる**直接的選好最適化**が主流になりつつある。中でもDPO（Direct Preference Optimization）は、報酬モデルを明示的に学習せずにポリシーを直接最適化するエレガントなアプローチとして広く採用されている。

しかし、DPOには根本的な制約がある。

**シーケンス全体に1つのラベルを割り当てる**という設計だ。例えば「なぜAの方がBより良いか」という判断を、出力全体に対して1つのyes/noで処理する。これは粒度が粗すぎる——ある応答の前半は優れているが後半で劣る、といったケースを捉えられない。

また、最近の研究（TEPO、SPPOなど）がトークンレベルの選好最適化に挑んでいるが、理論的な基盤が弱く、設計の動機付けが不十分という問題があった。

TBPOはこの課題に**数学的に厳密な解答**を与える。

## TBPOの技術コア

### 1. トークンレベルBradley-Terryモデル

TBPOの第一歩は、選好モデルをシーケンスレベルから**トークンレベル**へ拡張することだ。

通常のBradley-Terryモデルでは：

$$P(y_w \succ y_l) = \frac{\exp(r(y_w))}{\exp(r(y_w)) + \exp(r(y_l))}$$

TBPOではこれをトークン単位に分解する：

$$P(y_w \succ y_l) = \prod_{t=1}^{T} \frac{1}{1 + \exp(-(s_t^w - s_t^l))}$$

ここで $s_t$ はトークン位置 $t$ でのスコア関数、$s_t^w$ は勝者側のトークン $y_w^{<t}$ までのプレフィックスに基づくスコア、$s_t^l$ は敗者側のスコアだ。

重要なのは、この積の形が**再帰的にシーケンスレベルの確率に集約**されること。つまり、トークンレベルで学習したものが、全体としての選好予測にも一貫して機能する。

### 2. Per-state重み $w_t$

しかし、単純に各トークンで比較するだけでは不十分だ。位置 $t$ での比較は、勝者側のプレフィックス $y_w^{<t}$ と敗者側のプレフィックス $y_l^{<t}$ という**異なる状態**から行われる。この不一致を補正するため、TBPOはper-state重み $w_t$ を導入する。

- **TBPO-Q**: $w_t = \frac{Z_\theta(y_l^{<t})}{Z_\theta(y_w^{<t})}$（パーティション関数の比）
- **TBPO-A**: $w_t = \exp(\text{KL}(\pi_\theta \| \pi_{\text{ref}})[y_w^{<t}] - \text{KL}(\pi_\theta \| \pi_{\text{ref}})[y_l^{<t}])$（KLベースラインの差）

この重みが本質的な役割を果たす——どちらの状態から比較しているかという文脈情報をスコアに反映させる。

### 3. Bregman比マッチング

ここがTBPOの理論的核だ。

DPOはロジスティック損失を最小化するが、これは暗に**特定の分布仮定**を置いている。TBPOはこれを一般化し、**Bregman divergenge**に基づく密度比推定フレームワークを構築する。

$$\min_\theta \sum_t \mathbb{E}_{(y_w, y_l)} \left[ h\left(\frac{R_\theta(y_w^{<t})}{R_\theta(y_l^{<t})}\right) \cdot w_t \right]$$

ここで $h$ はSBA（Scaled Basu's power divergence）凸関数、$R_\theta$ はモデルの密度比だ。

![fig2](/images/tokenratio-bregman-token-preference/fig2.png)
*図2: TBPOフレームワークの全体像。入力ペアからトークン分解を行い、TBPO-Q（Q関数）とTBPO-A（アドバンテージ関数）の二系統でスコアを計算し、per-state重みで補正後にBregman比マッチング損失で最適化する。*

#### SBAによる統一的定式化

SBAパラメータ $\alpha$ を調整することで、様々な既存手法が特殊ケースとして含まれる：

| $\alpha$ | 対応手法 |
|----------|----------|
| 1 | ロジスティック回帰（DPO） |
| 0 | KLIEP |
| -1 | LSIF |
| 0.5 | TBPOのデフォルト |

論文の定理4.3は、この目的関数を最小化することが**トークンレベル最適ポリシーの回復**と等価であることを証明している。また定理4.4は、実用的な訓練可能な損失形式（モデル比 $R_\theta$ のみに依存）を導出する。

### 4. TBPO-Q vs TBPO-A

**TBPO-Q**（Q-function variant）:
- スコアとしてQ関数を使用
- LMバックボーンに**stop-gradient**を適用し、軽量MLPヘッドのみを学習
- $w_t$ はパーティション関数の比で計算（K3推定器使用）

**TBPO-A**（Advantage variant）:
- スコアとしてアドバンテージ関数を使用
- **追加パラメータゼロ**——既存のポリシーモデルのみで動作
- $w_t$ はKLベースライン差で計算

実用的にはTBPO-Aの方がシンプルで導入しやすいが、理論的にはTBPO-Qの方が強い保証を持つ。

## 実験結果

### メインベンチマーク

Mistral-7B-InstructとLlama-3-8B-Instructをベースに、Open LLM Leaderboardの6つのベンチマーク（ARC-C, HellaSwag, MMLU, TruthfulQA, Winogrande, GSM8K）で評価。

| 方法 | Mistral 7B Avg | Llama 3 8B Avg |
|------|---------------|----------------|
| SFT | 58.80 | 66.06 |
| DPO | 59.19 | 67.31 |
| IPO | 58.85 | 67.26 |
| KTO | 59.07 | 67.27 |
| TIS-DPO | 60.50 | 69.56 |
| **TBPO-Q** | **60.95** | **70.50** |
| **TBPO-A** | 60.46 | 70.12 |

![fig3](/images/tokenratio-bregman-token-preference/fig3.png)
*図3: Llama 3 8BでのOpen LLM Leaderboard結果。6つのベンチマーク×7手法のグループ化棒グラフ。TBPO-Q（濃赤）が全体的に最高スコアを記録。*

TBPO-Qは両モデルで最高スコアを達成。特にLlama 3 8BではDPOに対して**+3.19ポイント**の改善。

### 多様性の分析

ここがTBPOの最も印象的な結果だ。

アライメント手法の典型的なジレンマとして、**品質と多様性のトレードオフ**がある。TIS-DPOのような手法は品質を向上させるが、モード崩壊（全ての応答が似通う現象）を引き起こす。

TBPOはこのトレードオフを**パレート最適**な形で解決する：

- **Win Rate**: TBPO-Qが**最も高い勝率**（対話品質の観点）
- **応答長**: 同時に**最も短い**応答を生成
- **多様性**: Distinct-1、予測エントロピー、Self-BLEUの全てで良好なバランス

TIS-DPOが過度に多様性を犠牲にするのに対し、TBPOは品質向上と多様性維持を両立している。

![fig4](/images/tokenratio-bregman-token-preference/fig4.png)
*図4: 左パネルはアライメント品質と応答長の散布図——TBPO-Qが最短の応答で最高の勝率を達成。右パネルは多様性メトリクスの比較——TBPOはTIS-DPOのモード崩壊を回避しつつ高品質を維持。*

## 考察

### DPO一般化の理論的意味

TBPOの最大の貢献は、実用面以上に**理論的な完成度**にある。

これまでのトークンレベル選好最適化（TEPOやSPPOなど）は、経験的な設計に基づくものであり、「なぜこの損失関数がよいのか」に対する数学的根拠が不十分だった。TBPOは**Bregman divergenge**という確立された数学的枠組みから出発し、目的関数の最適解が最適ポリシーを回復することを**定理として証明**している。

これは、選好最適化の分野において「経験的アプローチから理論的アプローチへの転換点」と位置づけられる。

### TBPO-Aの実用性

追加パラメータゼロという点で、TBPO-Aは特に実用的だ。既存のDPOパイプラインに対して、損失関数を差し替えるだけで導入できる。stop-gradientのオーバーヘッドも微小であり、DPOとの置き換えコストはほぼ無いに等しい。

### 応答長の短縮効果

TBPOが最短の応答長で最高品質を達成するという結果は、実用上非常に重要だ。推論コストは応答長に比例するため、品質維持（あるいは向上）とコスト削減を同時に実現できるからだ。

これはOPSDやReasonMaxxerなどの「推論の効率化」研究とも共鳴する——良いアライメントは、無駄を削る方向に作用するということだ。

| 手法 | レベル | 損失関数 | 理論基盤 |
|------|--------|----------|----------|
| DPO | シーケンス | ロジスティック | BTモデル |

| KTO | トークン | 非ペアロジスティック | 非最適 |

TBPOはトークンレベルの粒度と、定理レベルの理論的保証を**初めて同時に実現**した手法と言える。

## まとめ

TBPOは、選好最適化の分野に以下の貢献をもたらした：

1. **理論的統合**: Bregman divergengeに基づく統一的フレームワークでDPOを含む多数の手法を一般化
2. **トークンレベルの厳密な定式化**: Per-state重みによる文脈補正と、最適ポリシー回復の定理証明
3. **実用性**: TBPO-Aは追加パラメータゼロ、TBPOは両モデルで最高スコア
4. **品質-多様性の両立**: TIS-DPOのモード崩壊問題を回避しつつ、最高品質を達成
5. **効率性**: 最短の応答長で最高品質——推論コストの削減にも貢献

アライメント研究が「いかに学習を安定させるか」から「いかに学習信号を精緻化するか」へ移行しつつある中、TBPOは後者の方向性において**理論と実践の両面で強力な指針**を提供している。

今後の展開として、RLVR（強化学習による推論訓練）との組み合わせや、より大規模モデルでの検証が期待される。
