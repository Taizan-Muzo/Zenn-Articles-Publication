# CIPO: 失敗から学ぶ——RLVRのための修正指向ポリシー最適化

## TL;DR

- RLVR（検証可能報酬付き強化学習）はスパースな二値報酬と弱い信用割当に悩まされ、失敗軌跡の有用情報が未活用だった
- **CIPO**（Correction-Oriented Policy Optimization）は、オンポリシーの失敗軌跡を修正指向の監督信号に変換し、標準RLVR目的関数と共同最適化する
- リスク回避型報酬整形、適応型リプレイ、難易度認識トラジェクトリ選好の3つのメカニズムを統合
- 数学推論6ベンチマーク・コード生成2ベンチマークでGRPOを一貫して上回り、pass@32で+6.12%の改善
- 4Bモデルの修正能力が72Bモデルに肉薄する結果をCriticBench/DebugBenchで実証

## 1. 問題の背景：RLVRの「失敗の無駄」

強化学習と検証可能報酬（RLVR）は、大規模言語モデルの推論能力を向上させる上で極めて有効なパラダイムとして確固たる地位を築いている。しかし、このパラダイムには根本的な弱点がある。

RLVRでは、モデルが生成した回答は「正解(1)」か「不正解(0)」の二値報酬しか得られない。GRPOなどのグループ相対ポリシー最適化手法では、各プロンプトに対してN個の軌跡をサンプリングし、グループ内の相対的優位性からアドバンテージを計算する。しかし、**失敗した軌跡はすべて一律の負のアドバンテージ**を受けるだけであり、エラーの種類や性質は一切区別されない。

![](/images/cipo-correction-policy-optimization/fig1.png)

上図の左側が示す通り、標準RLVRでは以下の3種類の失敗が同じ扱いを受ける：

1. **ニアミス（単純な計算ミスなど）**：ほんの少しの修正で正解に到達できる
2. **根本的な欠陥（アプローチ自体が間違っている）**：全体をやり直す必要がある
3. **部分的進捗（途中までは正しい）**：方向性は合っているが途中で崩れた

この「一律の負の抑制」は、特にニアミスや部分的進捗に含まれる豊富な学習信号を無駄にしている。CIPOの核心アイデアはシンプルだ——**失敗を抑制ではなく、方向性のある指導信号に変換する**。

## 2. 手法の詳細

### 2.1 CIPOの全体フレームワーク

CIPOの基本的な考え方は、モデル自身の失敗軌跡を「修正の種」として再利用することである。具体的には以下のステップで動作する：

**ステップ1：ベースロールアウト**
通常のRLVRと同様に、プロンプト $x$ に対してG個の応答をサンプリングし、二値報酬 $R(x,y) \in \{0,1\}$ で評価する。

**ステップ2：リプレイサンプルの構築**
失敗軌跡（成功軌跡も一部含む）を適応的に選択し、以下のような修正プロンプトを構築する：

```
{元のプロンプト x}
Below is a candidate solution from a large language model (correctness unknown):
<candidate_solution>
{候補解 y_cand}
</candidate_solution>
Please refer to this solution and provide your solution.
```

重要な設計選択として、候補解の正誤は「unknown（不明）」と提示する。正誤情報をリークするとRL目的自体が損なわれるためだ。

**ステップ3：修正ロールアウト**
構築した修正プロンプトから再度G個の応答をサンプリングし、整形報酬（shaped reward）で評価する。

**ステップ4：共同最適化**
ベース軌跡と修正軌跡を統合してポリシー更新を行う。

### 2.2 リスク回避型報酬整形（最重要コンポーネント）

CIPOの中で最も重要なコンポーネントは、リスク回避型報酬整形である：

$$R_{risk}(x, y, y') = R(x, y') - \lambda_{risk} \cdot \mathbb{I}[R(x,y)=1 \wedge R(x,y')=0]$$

これは一見シンプルだが、**「正解 → 不正解」の能力退化を明示的に抑制**する。条件付き軌跡 $y$ が正解なのに新しい応答 $y'$ が不正解になる場合、ペナルティ $\lambda_{risk}$ を課す。

アブレーション実験で、このコンポーネントを除去すると平均精度が**-6.97%**と最も深刻な悪化を示した。これは、修正訓練が「正解を忘れる」リスクを本質的に抱えており、それを抑え込む仕組みが不可欠であることを示している。

### 2.3 適応型リプレイメカニズム

成功軌跡と失敗軌跡のリプレイ比率 $\rho_t \in [\rho_{min}, \rho_{max}]$ を動的に調整する。モデルがリサイクルした成功サンプルを保持できているかどうかに基づき、以下のルールで更新：

$$\rho_{t+1} = \text{clip}[\rho_t(1 + w_1(R^* - R_t) + w_2 \max(0, R_{t-1} - R_t) + w_3 \min(c_t, 3))]$$

- $R_t$：成功軌跡からの修正の平均整形報酬
- $R^*$：目標保持水準（ハイパラ、0.80）
- $c_t$：連続低パフォーマンスカウンタ

固定1:1比率に比べ**+4.19%**の改善を確認。探索と利用の動的バランスが重要である。

### 2.4 難易度認識トラジェクトリ選好

ヴィゴツキーの「最近接発達領域（Zone of Proximal Development）」に着想を得て、中程度の難易度のプロンプトを優先的にリプレイする：

$$\mathcal{X}_{med} = \{x \in \mathcal{D} \mid \delta_{low} \leq \hat{P}(x) \leq \delta_{high}\}$$

経験合格率 $\hat{P}(x)$ が $\delta_{low} = 3/8$ 〜 $\delta_{high} = 6/8$ の範囲に入るプロンプトを優先する。あまり簡単すぎても（学習信号が薄い）、難しすぎても（修正成功確率が低すぎる）効果的でないため、この「スイートスポット」が鍵となる。

## 3. 実験結果

### 3.1 数学推論（Qwen3-4B）

![](/images/cipo-correction-policy-optimization/fig2.png)

6つの数学推論ベンチマーク（AIME24, AIME25, AMC23, MATH500, Minerva, Olympiad）で評価。CIPOは以下の結果を達成：

| メソッド | 平均精度 | AIME24 | AIME25 |
|----------|---------|--------|--------|
| GRPO (BS=128) | 59.83% | 42.08% | 35.73% |
| GRPO (BS=256) | 59.66% | 40.42% | 33.44% |
| PRIME | 62.88% | 48.65% | 40.42% |
| **CIPO** | **64.38%** | **47.50%** | **44.90%** |

GRPO(BS=128)に対して平均+4.55%、AIME25では+9.17%という大きな改善。計算量を2倍にしたGRPO(BS=256)に対しても+4.72%で勝利しており、**同じ計算予算で大幅な性能向上**が確認された。

### 3.2 pass@K：本質的推論能力の拡大

![](/images/cipo-correction-policy-optimization/fig4.png)

pass@Kの結果が特に重要だ。CIPOはpass@32で平均85.56%を達成し、GRPOの79.44%に対して**+6.12%**の改善を示した。pass@Kの改善は、モデルが既存の正解の確率質量を再分配しているだけでなく、**本質的な推論能力を拡大**していることを示す有力な証拠である。

### 3.3 修正能力：4Bモデルが72Bに肉薄

CriticBench（修正・批評能力評価）の結果が衝撃的だ：

| モデル | 修正精度 |
|--------|---------|
| Qwen2.5-72B-Instruct | 74.69% |
| Qwen3-32B | 75.85% |
| Initial (Qwen3-4B) | 68.75% |
| +GRPO | 70.43% |
| **+CIPO** | **72.82%** |

CIPO適用後の4Bモデルは、GRPO訓練の4Bモデルに対して+7.74%の修正精度向上を達成し、72BのQwen2.5-Instruct（74.69%）に**肉薄**している。

さらにDebugBench（コードデバッグ）では、CIPO適用後のSeed-Coder-8Bが平均64.99%を記録し、**Claude-Sonnet-4（65.58%）にほぼ匹敵**する結果を示した。

### 3.4 アブレーション：各コンポーネントの寄与

![](/images/cipo-correction-policy-optimization/fig3.png)

全5つのコンポーネント除去実験の結果：

| 除去コンポーネント | 精度変化 |
|------------------|---------|
| Risk-Averse Shaping | **-6.97%**（最深刻） |
| Adaptive Control | -4.19% |
| On-Policy Replay (BS=256) | -3.94% |
| On-Policy Replay (BS=128) | -3.91% |
| Difficulty-Aware Preference | -3.40% |

リスク回避型報酬整形が圧倒的に重要であることが定量的に証明された。また、オンポリシーリプレイ（-3.91%）とオフラインリプレイ（BS=128, -3.91%）の差がほぼないように見えるが、計算量を考慮するとオンポリシー版が優位に立つ。

## 4. 考察

### 「失敗の種類を区別できない」というRLVRの根本的弱点に対する的確な回答

CIPOの最も面白い点は、失敗の種類を明示的に区別しなくても、**修正サンプリングの成功確率が自然にエラーの深刻度を反映する**ことだ。ニアミスな失敗は修正成功確率が高く、根本的な欠陥は低い——この確率差が暗黙的な信用割当として機能する。

これはProcess Reward Model（PRM）を使った段階的評価とは異なるアプローチであり、追加の注釈やモデルを一切必要としない点が実用的価値が高い。

### pass@Kの改善が意味するもの

「確率質量の再分配」という批判は、RLHF/RLVR分野でしばしば議論される問題だ。しかしCIPOのpass@K改善は、新しい正解解法の発見を促進していることを示唆している。モデルは単に「正解を覚える」のではなく、「どうすれば正解にたどり着けるか」を学習している。

### オンポリシー性の重要性

CIPO-Offline（初期化時の軌跡のみリプレイ）と比較してオンポリシーバージョンが+3.91%優れることは、**動的に変化する失敗モードに追従する**ことの重要性を示している。訓練が進むにつれてモデルが犯すエラーの性質も変化し、それに応じた修正監督が必要不可欠だ。

## 5. 関連研究との位置づけ

- **PRM（Process Reward Model）**：段階的正解性を評価するが、追加の注釈コストが必要。CIPOは外部信号なしに暗黙的な段階的評価を実現
- **HER（Hindsight Experience Replay）**：失敗したゴールを実績で置き換えるRL手法。CIPOは失敗軌跡を「修正のアンカー」として再利用する点で思想的に類似
- **SDPO**：フィードバック条件付き教師を構築する手法。外部フィードバックが必要な点でCIPOとは異なる
- **InsightReplay**（2026-05-15紹介）：長CoTでの洞察のアテンション減衰問題に焦点を当てた手法。CIPOとは対照的に、成功軌跡の有用部分に注目
- **ResRL**（2026-05-06紹介）：SVDによる負サンプル分離。CIPOは分離ではなく変換で失敗を活用する

## 6. まとめ

CIPOは「失敗をどう扱うか」というRLVRの根本的な弱点に対し、シンプルだが強力な回答を提示した。失敗軌跡を一律に抑制するのではなく、修正指向の監督信号に変換し、リスク回避・適応リプレイ・難易度認識の3つのメカニズムで効果的に学習する。

特に印象的なのは、4Bモデルの修正能力が72Bモデルに迫るというスケールを超えた結果だ。これは「大きなモデルを作る」こととは別の軸で性能を引き出せる可能性を示しており、リソース制約の厳しい実践環境での応用価値が高い。
