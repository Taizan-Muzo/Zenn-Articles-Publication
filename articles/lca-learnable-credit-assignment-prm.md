# TL;DR

Process Reward Model（PRM）はLLMの推論能力を強化するが、ステップごとの注釈コストが高い。結果のみからPRMを訓練する手法は**信用割当（Credit Assignment）**問題を抱える──誤ったステップにどの程度の責任を帰属させるかが未解決だった。

北京大学のJiaらが提案する**LCA（Learnable Credit Assignment）**は、この問題をMultiple Instance Learning（MIL）として定式化し、**最弱リンク原理**（"a reasoning chain is as strong as its weakest link"）に基づく**Softmax-Weighted-Sum（SWS）Pooling**を導入する。理論的にはBayes一致性を証明し、実験的にはProcessBenchで既存SOTAを最大+21.7%上回る結果を達成した。

# 背景

## PRMと信用割当のジレンマ

LLMの推論は、数学やコーディングといった領域で飛躍的な進歩を遂げている。その推論品質をさらに引き上げる鍵として、Process Reward Model（PRM）が注目されている。PRMは推論チェーンの各ステップの正しさを評価し、テスト時スケーリング（Best-of-N）や強化学習での報酬信号として機能する。

PRMの訓練には二つのアプローチがある：

1. **ステップ監督型**：各ステップが正しいかを人間が注釈 → 高精度だがコストが極めて高い
2. **結果監督型**：最終回答が正しいかだけを利用 → スケーラブルだが信用割当問題が生じる

結果監督型の難しさは直感的に理解できる。答えが間違っていた場合、5ステップの推論チェーンのどこで失敗したのか？全ステップに均等に責任を負わせるべきか、それとも特定のステップに集中させるべきか？

## 既存の信用割当戦略

これまでに二つの主要なアプローチが提案されてきた：

### Uniform Assignment（均一割当）
全ステップに等しい重みを割り当て、ステップスコアを**加算**で集約して軌跡スコアを得る。代表例としてChen et al. (2025)やYuan et al. (2025)。

```
p_hat = (1/T) * sum(p_t)
```

問題点：間違ったステップと正しいステップを区別できない。

### Causal Assignment（因果割当）
各ステップのスコアを、そのステップ以降の成功確率（因果効果）として推定する。代表例としてMathShepherd、OmegaPRM、RStar-Math、SCAN。

```
p_hat(t) = E[outcome | perturb step t]
```

問題点：ステップ自身の正しさではなく、下流への影響を測るため、本質的に正しいステップでも低スコアになり得る。

### 共通の欠陥

**どちらの戦略も、ステップの正しさに信用をアンカーしていない**。PRMの本来の目的は「どのステップが間違っているかを特定すること」であるにもかかわらず、既存手法はステップの正しさとは無関係な指標で信用を配分している。

# 方法：LCA

## 最弱リンク原理（Weakest Link Assignment）

LCAの基盤となる洞察はシンプルだ：

> **推論チェーンは最弱のリンクと同じ強さしかない**

正式には、軌跡の正しさはステップの正しさの最大値で定義される：

```
y(tau) = max{y(tau_1), ..., y(tau_T)}
```

ここで `y(tau_t) = 1` はステップtが**間違っている**ことを示す。つまり、一つでも間違ったステップがあれば軌跡全体が間違いとなる。これはPRM評価で広く使われる**首錯検出（First Error Detection）**基準と完全に一致する。

## MIL形式化

この問題をMultiple Instance Learning（MIL）として定式化する：

| MIL概念 | PRMにおける対応 |
|---------|----------------|
| Bag（バッグ） | 完全な推論軌跡 |
| Instance（インスタンス） | 各推論プレフィックス（ステップ1〜tまで） |
| Bag label | 軌跡の最終結果（正/誤） |
| Instance label | プレフィックスの正誤 |

バッグラベルはインスタンスラベルの最大値となる。これはMILの**Standard Multi-Instance Assumption**と完全に一致する。

### 推論プレフィックスの特有な性質

ただし、一般のMILとは異なり、推論プレフィックスには三つの特有な性質がある：

1. **自己回帰性**：後のプレフィックスは前のプレフィックスを包含する
2. **入れ子性**：全プレフィックスが入れ子構造を持つ
3. **単調性**：エラーは蓄積するのみで消失しない

これらにより、インスタンス間の**強い依存性と冗長性**が生じる。一般のMILプーリング手法（max poolingやattention pooling）はこの条件下ではうまく機能しない。

## SWS Pooling（Softmax-Weighted-Sum Pooling）

これらの課題に対処するため、LCAは**Softmax-Weighted-Sum（SWS）Pooling**を導入する：

```
SWS_alpha(p) = sum_t (w_t * p_t)

w_t = exp(p_t / alpha) / sum_j exp(p_j / alpha)
```

### 温度パラメータαの役割

- **α → 0**：max poolingに近づく。間違ったステップに信用を集中
- **α → ∞**：mean poolingに近づく。全ステップに均等に信用を分散

重要なのは、**全ての w_t が厳密に正の下界を持つ**こと。max poolingとは異なり、どのステップも非ゼロの勾配を受け取る。同時に、Softmaxの重みは自然に最弱リンク原理を encode する──高スコア（間違いの可能性が高い）のステップが自動的に大きな重みを獲得する。

## 学習アルゴリズム

LCAの学習は極めてシンプルだ：

1. **Forward Pass**：PRMが軌跡の全プレフィックススコア `f(tau) = (p_1, ..., p_T)` を一括出力
2. **SWS Pooling**：スコアをSWSで集約して軌跡スコア `p_hat(tau)` を取得
3. **Loss**：軌跡レベルの交差エントロピー損失で訓練

```
L = (1/K) * sum CE(SWS_alpha(p), y)
```

一つの追加技術：**Softmax重み w_t にstop-gradientを適用**して訓練を安定化させる。これにより、重みの更新と分類器の更新が適切に分離される。

## 理論的保証：Bayes一致性

LCAの最も強力な理論的貢献は、**Bayes一致性の証明**である。

### 定理3.1（唯一の大域的最小値）
M仮定の下で、十分小さい温度 α > 0 に対して、損失関数 L は真のラベル関数 `f*(x) = y(x)` によって**唯一に最小化**される。

### 定理3.2（確率収束）
損失 `{L(f_l)}` が最小値 `L(f*)` に収束するならば、分類器 `{f_l}` は**確率収束**してBayes最適分類器 `f*` に至る。

### M仮定とは
負インスタンスが正バッグに出現する確率が、任意に大きくなることはないという仮定。直感的には「明らかに間違ったステップだけを含む軌跡が正解することはない」という条件で、推論タスクでは自然に満たされる。

## 反省・バックトラックへの拡張

LCAは単純な線形推論チェーンだけでなく、**反省（Reflection）やバックトラック（Backtracking）**を含む推論軌跡にも拡張可能である。付録Eで示されている通り、木構造の推論軌跡に対してもMIL形式化が適用でき、SWS Poolingを階層的に構成できる。

## RQ3：温度αの影響

![Temperature Effect](/images/lca-learnable-credit-assignment-prm/fig3.png)

温度αは単純なスケーリングパラメータではなく、**タスクの難易度に応じて最適値が異なる**：

- **シンプルなタスク（GSM8K）**：低いαが有利。エラー箇所に信用を集中させる方が精度が高い
- **難しいタスク（OlympiadBench, Omni-MATH）**：高いαが有利。複数の間違いを同時に検出する必要があるため、信用を分散させる方が有効

この洞察は実用的にも重要だ。タスクに応じてαを調整するだけで、LCAの性能を最適化できる。

## RQ4：プーリング手法の比較

SWS Pooling vs. その他のMILプーリング手法（instance pooling、embedding pooling、attention pooling、conjunction pooling等）の比較でも、SWSが一貫して最高性能を示す。特にα=0.2の設定では、全教師ありPRMの上限に近い性能に到達する。

# 考察

## 最弱リンク原理の妥当性

LCAの成功は、最弱リンク原理の妥当性を裏付けている。推論が連鎖的プロセスである以上、一つの誤りが最終結果を破綅させるのは数学的に正しい。このシンプルな原理を信用割当の指導原理とすることで、既存の複雑な因果推定ベースの手法を大幅に上回れるのは、AI研究における「正しい抽象化の力」の一例と言える。

## MIL形式化の優雅さ

結果監督PRMをMILとして再定式化するのは、一見すると些細な見方の変更に思えるかもしれない。しかし、これによって以下の利点が同時に得られる：

1. **理論的基盤**：MILの豊富な理論的結果（Bayes一致性など）をPRMに流用可能
2. **既存手法の統一的理解**：均一割当はmean pooling、因果割当はmax poolingに近い操作として解釈できる
3. **新手法の設計空間**：SWSのように、推論特有の性質に合わせたプーリングを設計できる

## 実用上の意義

LCAは**完全に結果監督型**であり、ステップレベルの注釈を一切必要としない。これは以下の実用的な利点を意味する：

- **スケーラビリティ**：最終回答の正誤判定のみで訓練可能 → 大規模データセットに適用可能
- **汎用性**：任意の領域の推論タスクに適用可能（数学だけでなく、コーディング、論理推論等）
- **効率性**：追加の推論やモンテカルロサンプリングを必要としない

## 残された課題

論文も正直に以下の限界を認めている：

1. **全教師ありPRMとのギャップ**：ステップ注釈付きで訓練したPRMにはまだ及ばない
2. **ノイズのあるラベル**：LLMが「間違った推論で正解にたどり着く」場合があり、軌跡ラベルに固有のノイズが含まれる
3. **RL統合の課題**：PRMを強化学習の報酬として使う際のreward hacking問題は未解決

# 関連研究

## 結果監督PRM
- **MathShepherd** (Wang et al., 2023)：因果割当の先駆者。モンテカルロ推定で各ステップの偽ラベルを生成
- **OmegaPRM** (Yuan et al., 2025)：因果割当を改良し、自己一致推定を導入
- **RStar-Math** (Zhang et al., 2025)：推論とPRM訓練の相互強化
- **SCAN** (Li et al., 2025)：自己対比による信用割当

## MILとプーリング
- **Attention MIL** (Ilse et al., 2018)：attention機構に基づくMILプーリング
- **DSMIL** (Zhu et al., 2021)：二段階プーリング手法
- LCAのSWSはこれらと異なり、推論プレフィックス特有の依存性に特化して設計されている

## PRMの応用
- **Best-of-N** (Lightman et al., 2023)：PRMによるテスト時スケーリング
- **RLVR**（本シリーズで度々取り上げている手法群）：PRMをRLの報酬として利用

# まとめ

LCAは、結果監督PRMの根本的な課題である信用割当問題に対し、最弱リンク原理に基づくエレガントな解決策を提示した。

**三つの主要な貢献**：

1. **洞察**：既存手法の共通欠陥を特定し、最弱リンク原理を原則的な代替案として提案
2. **技術**：MIL形式化 + SWS Poolingで信用割当と報酬モデリングを共同学習。Bayes一致性を数学的に証明
3. **実験**：ProcessBenchとBest-of-Nスケーリングで一貫してSOTAを達成

「推論チェーンは最弱のリンクと同じ強さしかない」という原理は一見当然のように聞こえるが、それを形式的に定式化し、実用的なアルゴリズムに落とし込み、かつ理論的保証まで付与した点で、LCAは結果監督PRM研究における重要なマイルストーンと言える。

PRMがLLM推論の評価・改善においてますます重要になる中、ステップ注釈なしで高品質なPRMを訓練できる手法は、実用上の価値も大きい。今後の展開として、RLVRパイプラインへの統合や、反省・バックトラックを含む複雑な推論軌跡への対応が期待される。
