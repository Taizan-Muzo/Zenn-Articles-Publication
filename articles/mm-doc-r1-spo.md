## TL;DR

- 長文書VQAにおいて従来のRAGが苦手とする**複数証拠を跨ぐ多ホップ推論**を、多轮Agentワークフローで解決
- **SPO（Similarity-based Policy Optimization）**：多轮RLでGRPOが抱える「初期状態の基線を全中間状態に流用する」偏差を、軌跡間の意味的類似度で重み付けした基線推定で修正
- MMLongBench-Docで最強基線比**+10.4%**（ACC 38.4→49.7%）、SPO>GRPOは+5.0〜6.1%
- Planner→Seeker→Answer Agentの3フェーズ構成；BM25検索＋Qwen2.5-VLページ解析ツールを反復呼び出し
- 論文：arXiv:2604.13579（Fudan大学・上海Singularity AI、2026年4月15日）

## なぜ長文書VQAは難しいのか

PDFで100ページを超えるような技術文書・財務報告書・学術論文に対して「複数箇所の情報を組み合わせて答えを出す」タスクは、現在のRAGシステムにとって本質的に難しい。

理由はシンプルだ。**シングルパス検索**は1回のベクトル類似検索でTop-Kチャンクを取ってくるが、「表Aの数値と本文第3節の条件を組み合わせる」ような多ホップ推論では、最初の検索で必要な証拠をすべて捕捉できない。

さらに長文書は**視覚情報**（グラフ、表、図）が重要な役割を果たす。テキストのみのRAGでは、画像に埋め込まれた数字や構造を見落とす。

従来手法の最高スコア（M3doc RAG）はMMLongBench-DocでACC 38.4%。人間が65.8%を出すのに対して、まだ大きな差がある。

## MM-Doc-R1のフレームワーク

MM-Doc-R1は大きく3つのフェーズで動作する。

### Phase 1：文書前処理

ユーザーの質問を受け取る前に、文書全体を**Doc2X**でOCR処理し、表・図・テキストをMarkdown形式に変換する。同時に目次（TOC）を生成し、ページごとのスクリーンショットを保持する。

### Phase 2：Planner（計画立案）

- 入力：ユーザー質問 ＋ 文書TOC ＋ 図表一覧
- 出力：最大10ステップの調査計画
- 使えるツール：**検索ツール**（BM25によるキーワード検索）と**読取ツール**（最大5ページ）

PlannerはGPT-likeな「まずアウトラインを立ててから実行する」思考様式で、どのページを読むべきか大局的な方針を作る。

### Phase 3：Seeker（情報探索）＋ Answer Agent（回答生成）

Seekerは多轮で動く核心コンポーネントだ。

各ターンで以下を繰り返す：
1. サブクエリを生成
2. 検索ツール（BM25 Top-10）またはページ読取ツール（VLM: Qwen2.5-VL-7B）を選択・実行
3. 得られた情報を評価。足りなければ次ターンへ
4. 十分な証拠が集まれば `<FINISH>` トークンを出力

最後にAnswer Agentが全ターンの出力をまとめて最終回答を生成する。

## SPO：GRPOの基線偏差を修正する

### GRPOの問題：初期状態の基線が全中間状態に適用される

多轮RLでGRPOを使う際の根本的な問題を理解しよう。

GRPOはグループ内のN本の軌跡$\{T_1, T_2, ..., T_N\}$について、それぞれの優位量を：

$$A_{\text{GRPO}}(T_i) = R(T_i) - \frac{1}{N}\sum_{j=1}^{N} R(T_j)$$

と計算する。全軌跡の平均報酬を「基線（ベースライン）」として引くわけだ。

シングルターン推論なら問題は少ない。しかし**多轮RLでは各ターンごとに軌跡が分岐する**。「ステップ1でページ3を読んだグループ」と「ステップ1でページ7を読んだグループ」は、初期状態が違うにも関わらず同じ基線を共有させられる。これが不正確な優位量推定につながり、訓練を不安定にする。

### SPOの解決策：相似度加重基線

SPOのアイデアはエレガントだ。**「自分と似た軌跡ほど、より正確な基線を与えてくれる」**という直感から出発する。

SPOでは優位量を：

$$A_{\text{SPO}}(T_i) = R(T_i) - \sum_{j=1}^{N} w_{ij} R(T_j)$$

と定義する。重み$w_{ij}$は軌跡の意味的類似度に基づく：

$$w_{ij} = \frac{\text{sim}(\text{emb}(T_i),\, \text{emb}(T_j))}{\sum_{k=1}^{N} \text{sim}(\text{emb}(T_i),\, \text{emb}(T_k))}$$

軌跡の埋め込みには冷凍された**BGE-M3**を使用。類似度は余弦類似度で計算する。

結果として、「同じ文書の同じ部分を読んだ近い軌跡」からは強い基線信号を受け取り、「全く違う探索経路をたどった軌跡」には低い重みしか与えない。これにより各中間状態に対してより適切な基線を推定できる。

## 実験結果

### MMLongBench-Doc での性能比較

| 手法 | ACC | F1 |
|------|-----|----|
| BM25（シングルパス） | 36.4 | 31.0 |
| BGE-M3（シングルパス） | 39.3 | 34.8 |
| M3doc RAG（最強基線） | 38.4 | 36.7 |
| MM-Doc-R1 + GRPO (Qwen3-4B) | 39.9 | 36.3 |
| **MM-Doc-R1 + SPO (Qwen3-4B)** | **46.0** | **41.2** |
| MM-Doc-R1 + GRPO (Qwen3-8B) | 44.7 | 41.9 |
| **MM-Doc-R1 + SPO (Qwen3-8B)** | **49.7** | **46.1** |
| 人間の正解率 | 65.8 | 66.0 |

SPO (Qwen3-8B)は最強基線（M3doc RAG）を**+10.4%**上回り（38.4→49.7%）、同じモデルでGRPOを使うより**+5.0%**高い（44.7→49.7%）。

4Bモデルでは差がさらに大きく、GRPO比**+6.1%**（39.9→46.0%）。これは小さいモデルほど、適切な基線推定が訓練効果に影響しやすいことを示唆する。

### カテゴリ別分析（Qwen3-8B）

SPOはすべてのカテゴリで改善を達成している。なかでも：
- **図表（Chart）**: 35.7→40.8%（+5.1%） ← VLMの情報を適切に活用した証拠
- **表（Table）**: 49.9→52.8%（+2.9%）
- **画像（Figure）**: 28.3→35.9%（+7.6%） ← 特に大きな改善

### 消融実験

各コンポーネントを除いた実験では：

| 設定 | ACC | F1 |
|------|-----|----|
| MM-Doc-R1 完全版 | 45.7 | 41.5 |
| TOCなし | 44.3 | 39.9 |
| ページOCRなし | 43.4 | 38.7 |
| VLM読取なし | 42.1 | 37.3 |

VLMによるページ読取が最も重要なコンポーネントで、除去すると-3.6%。視覚情報の活用が長文書理解の鍵であることを確認。

## 考察

### SPOが解決する問題の本質

GRPOの問題をもう少し深く掘り下げたい。多轮RLでは**各ターンのトランジションが異なる情報に基づいて行われる**。Seekerが「ステップ1でページ5を読んで有益な情報を得た」軌跡と、「ページ12を読んで無関係な情報を得た」軌跡では、ステップ2以降の最適行動が根本的に異なる。

GRPOはこれらを同じグループに入れて同じ基線で比較するが、SPOは「ページ5を読んだ軌跡の仲間から基線を計算する」ことで、より公平な比較を実現する。

これはRLの**信用配分（credit assignment）**問題の多轮版への取り組みとも言える。単一ターンのTEPO（トークンレベル）とは異なるが、「どのエピソードと比較するか」という基線推定の精度を高めるという点で、同じ方向性を向いている。

### 長文書理解の難しさとマルチモーダルの重要性

消融実験でVLM読取が最も重要なコンポーネントだったことは興味深い。財務報告書・技術仕様書・学術論文では、**核心情報がグラフや表の形で視覚的に提示される**ケースが多い。テキスト埋め込みだけでは表内の数値関係やグラフの傾向を捉えきれないため、VLMによる直接的なページ解析が不可欠になる。

これは長文書AIの実用化に向けた重要な示唆だ。

### 課題と今後の方向性

現状の課題として：
- Seekerの最大反復回数がハイパーパラメータとして固定されており、文書の難易度に適応しない
- BGE-M3の埋め込み計算がオーバーヘッドを追加する（小さい代価だが）
- 日本語・多言語文書への適用はまだ未検証

今後の発展としては、SPOをより一般的な多轮RLタスク（コード実行、ツール利用エージェント）に拡張する可能性が高い。

## 関連研究との位置づけ

- **GRPO（DeepSeek, 2024）**: SPOの直接のベースライン。単一ターン推論では効果的だが、多轮RLで基線偏差が問題に
- **SPPO（ACL 2026）**: シングルターン×Contextual Bandit再定式化のアプローチ。MM-Doc-R1とは問題設定が異なる
- **TEPO（2025/2026）**: トークンレベルの信用配分を改善。SPOの「エピソードレベルの基線改善」と相補的
- **M3doc RAG**: 同タスクの最強基線。単純RAGの限界を示す対照として機能
- **DocVQA/MMLongBench**: 長文書理解の標準ベンチマーク群。MM-Doc-R1はMMLongBench-Docに特化

## まとめ

MM-Doc-R1は2つの問題を同時に解決した。

1. **アーキテクチャ面**: シングルパスRAGの限界を、Planner→Seeker→Answerの多轮Agentフローと視覚ページ読取ツールで突破
2. **学習面**: 多轮RLにおけるGRPOの基線推定偏差を、軌跡類似度加重のSPOで修正

MMLongBench-Docでの**+10.4%**という数字は、方向性が正しいことを示している。しかしまだ人間の65.8%に対して49.7%と差がある。長文書の完全な理解には、より深い多ホップ推論とより正確な視覚情報抽出が必要で、これは今後の研究の豊かな土壌だ。

長文書を「読む」から「考えながら読む」へ。その転換を多轮RLで実現しようとするMM-Doc-R1の試みは、実用的なドキュメントAIの未来を先取りしている。
