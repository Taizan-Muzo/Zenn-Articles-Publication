# Agentic RLの「見えない崩壊」：エントロピーでは検出できないTemplate Collapse

> 多ターンLLM Agentの強化学習訓練中、モデルは「推論できているつもり」で入力無関係なテンプレート出力に陥る。既存のエントロピーベースの指標ではこれを検出できない。RAGEN-2は、相互情報量（Mutual Information）による診断とSNR-Aware Filteringによる解決策を提案する。

## TL;DR

- Agentic RL訓練中、**エントロピーは安定しているのに推論が実質的に崩壊する**「Template Collapse」が発生する
- エントロピーは「同一入力内の多様性」しか測れない。真に重要なのは**「入力間の区別可能性（MI）」**
- MIはタスク性能とのSpearman相関 **r=+0.39**、エントロピーは **r=-0.14**（ほぼ無相関）
- 崩壊の原因は**SNR（信号対雑音比）の低下** → 正則化項が支配的になり、入力依存性が消失
- **SNR-Aware Filtering**（報酬分散上位のみ使用）だけで、複数タスクで一貫した性能向上を達成

## なぜこの問題が重要か

DeepSeek-R1以降、LLMの推論能力を引き出すために強化学習（RL）を使うアプローチが主流になっている。特にマルチターンのAgentタスク（計画立案、ウェブナビゲーション、コード実行など）では、モデルが自らの行動を環境と相互作用させながら学習する「Agentic RL」が鍵となる。

しかし、Agentic RLの訓練は**本質的に不安定**だ。そして、この不安定性は従来の指標では**見えにくい**。

この論文が指摘する「Template Collapse」は、まさにその「見えない崩壊」だ。

## Template Collapseとは何か

### 問題の本質

Template Collapseを一言で言えば：

> **「出力が見た目は多様だが、実は入力に応じて変化していない」状態**

具体例で説明しよう。3つの異なる入力（Input A, B, C）に対して、モデルはそれぞれ異なる回答を出力する。**一見すると多様に見える**。エントロピーを計算しても高値が出る。

しかしよく見ると、A, B, Cそれぞれの回答の「構造」は同じだ。

```
Input A: "Let me think... Step1: analyze → Step2: plan → Step3: execute"
Input B: "Let me think... Step1: analyze → Step2: plan → Step3: execute"  
Input C: "Let me think... Step1: analyze → Step2: plan → Step3: execute"
```

ステップの中身はわずかに異なるかもしれないが、**推論の「型」が固定化**されている。入力Aの問いが数学であれ、入力Cの問いがナビゲーションであれ、同じ枠組みに押し込めている。

これがTemplate Collapseだ。

### なぜエントロピーでは検出できないか

エントロピー（$H(Z)$）は**「同じ入力に対して何回サンプリングしても多様な出力が出るか」**を測る。Template Collapseの状態では、同一入力内でも出力にランダムな揺らぎがあるため、エントロピーは高いまま残る。

つまり、エントロピーが高い ≠ 入力に応じた推論ができている、ということだ。

## 診断手法：相互情報量（Mutual Information）

### 推論品質の分解

RAGEN-2は推論品質 $Z$ を2つの成分に分解する：

$$Z = H(Z|X) + I(X; Z)$$

- $H(Z|X)$: **条件付きエントロピー** — 同一入力内の多様性（＝エントロピー）
- $I(X; Z)$: **相互情報量** — 入力間の区別可能性（＝入力に応じて推論が変化する度合い）

この分解が示唆するのはシンプルだ：推論が高品質であるためには、**両方が重要**だが、エントロピーだけを見て満足していると$I(X;Z)$の低下を見逃す。

### MI代理指標の設計

実際のMIは推論空間が大きすぎて直接計算できない。RAGEN-2は**バッチ内交差スコアリング（in-batch cross-scoring）**を用いたMI代理指標を提案する：

- **Retrieval-Acc**: バッチ内の異なる入力から正しい入力を推論だけで特定できるか
- **MI-ZScore-EMA**: 出力の統計量の入力間分散をモニタリング

これらは追加のモデルや計算リソースなしに、訓練中にオンラインで計算可能だ。

![MI vs エントロピーの相関比較](/images/ragen2-template-collapse/fig2_mi_vs_entropy.png)
*Fig.2: MIとタスク性能の強い正相関 vs エントロピーと性能の弱い負相関*

### 実証結果

Sokoban（計画タスク）での実験では：

| 指標 | タスク性能とのSpearman相関 |
|------|---------------------------|
| エントロピー | **-0.14**（弱い負相関） |
| 相互情報量（MI） | **+0.39**（中程度の正相関） |

MIは性能低下に**先行的に反応**するため、早期警告指標として機能する。エントロピーが安定している間に、MIはすでに低下し始めている。

## 崩壊の原因：SNRメカニズム

### なぜTemplate Collapseが起きるのか

RAGEN-2は、Template Collapseを**信号対雑音比（Signal-to-Noise Ratio, SNR）**の観点から説明する。

RL訓練の勾配更新は大きく2つの成分から成る：

1. **タスク勾配**: 報酬から得られる「タスクを解くための」勾配
2. **正則化勾配**: KL分散ペナルティやエントロピーボーナスなどの「分布を制約する」勾配

問題は、**報酬分散（reward variance）が低いプロンプト**では、タスク勾配が弱くなることだ。どの推論パスをとっても報酬が似たような値になるなら、モデルに「これが正解だ」と伝える信号が弱い。

タスク勾配が弱くなると、正則化勾配が相対的に支配的になり、更新が**「分布を均す方向」**に偏る。結果として、異なる入力に対する推論の差異がどんどん消えていく。

$$\text{SNR} = \frac{\text{Var}[R]}{\text{Regularization Strength}} \xrightarrow{\text{low}} \text{Template Collapse}$$

![SNRメカニズムとフィルタリング戦略](/images/ragen2-template-collapse/fig3_snr_filtering.png)
*Fig.3: SNRの低下がTemplate Collapseを引き起こすメカニズムと、SNR-Aware Filteringによる解決*

## 解決策：SNR-Aware Filtering

### アルゴリズム

SNR-Aware Filteringは驚くほどシンプルだ：

1. **グループサンプリング**: 各プロンプト$p$について$G \geq 2$本の推論軌跡をサンプリング
2. **報酬分散の計算**: 同一プロンプト内での軌跡間報酬分散$\text{Var}[R_p]$を計算
3. **フィルタリング**: 分散上位Top-$p$のプロンプトのみを保持
4. **ポリシー更新**: フィルタされた高信号プロンプトのみで勾配更新

```
# 擬似コード
for prompt in batch:
    trajectories = sample(prompt, n=G)
    rewards = [evaluate(t) for t in trajectories]
    variance = np.var(rewards)
    
high_signal = select_top(variances, ratio=p)
update_policy(high_signal)  # 低信号プロンプトは使わない
```

### なぜこれで効くのか

報酬分散が高いプロンプトは、「良い推論と悪い推論の違いがはっきりしている」状態だ。つまり、**モデルにとって「学ぶべきこと」が多いプロンプト**だ。これらを優先的に使うことで：

- タスク勾配が強いサンプルに更新を集中できる
- 正則化勾配の支配を防ぐ
- 入力依存性（MI）を維持できる

## 実験結果

![各タスクでの性能向上](/images/ragen2-template-collapse/fig4_performance.png)
*Fig.4: SNR-Aware Filteringによる複数タスクでの一貫した性能向上*

### メインタスク結果

| タスク | Baseline | + Filtering | 向上率 |
|--------|----------|-------------|--------|
| Sokoban（計画） | 35.2% | 44.8% | **+27.3%** |
| MetaMathQA（数学） | 42.8% | 51.3% | **+19.9%** |
| WebNav（ナビゲーション） | 28.5% | 36.7% | **+28.8%** |
| Code Exec（コード） | 38.1% | 47.9% | **+25.7%** |

全タスクで20-30%程度の性能向上を確認。特に計画タスクとナビゲーションタスクで大きな改善が見られた。

### 汎用性の検証

SNR-Aware Filteringは以下の条件で有効性を確認している：

- **モデルスケール**: 0.5B ~ 7B+
- **RLアルゴリズム**: PPO, DAPO, GRPO, Dr.GRPO
- **タスク種別**: 計画、数学、ナビゲーション、コード、QA

特定のアルゴリズムやモデルに依存しない、汎用的な手法だ。

### 追加の知見

- **計算オーバーヘッド**: フィルタリング自体はほぼ無料。むしろ低信号サンプルを除外することで**訓練速度が向上**する
- **ハイパーパラメータ**: フィルタリング比率$p$は0.3-0.5程度で安定した結果
- **MIの推移**: フィルタリング適用後、MIの低下が緩和され、最終性能が向上

## 考察

### 「当たり前だけど誰も気づかなかった」感

この論文の最も大きな貢献は、新しいアルゴリズムの提案というより**「既存の診断指標が不十分だった」という事実の明示**にある。

エントロピーでRL訓練の安定性をモニタリングするのは、現在のLLMトレーニングコミュニティではほぼ常識だ。しかし、その「常識」が**致命的な見落としを生んでいた**という指摘はインパクトが大きい。

Template Collapseは、KL divergenceやエントロピーロスが安定して推移しているにもかかわらず、**モデルの推論能力が実質的に低下している**状態だ。訓練曲線だけ見ていると問題に気づかない。

### 情報理論の応用としての美しさ

MIによる分解 $Z = H(Z|X) + I(X;Z)$ は、情報理論の基本的な関係式だが、LLMの推論品質診断に応用する視点は新鮮だ。「多様性（エントロピー）」と「識別可能性（MI）」の2軸で推論を評価する枠組みは、今後のRL研究で広く使われる可能性がある。

### 実用性の高さ

SNR-Aware Filteringの実装は**数行のコード**で済む。既存のRLパイプライン（TRL, OpenRLHFなど）にプラグイン可能で、モデルアーキテクチャの変更も不要だ。この「低コストで大きな改善が得られる」という点は、実践的な価値が高い。

## 関連研究との位置づけ

- **RLVR（RL from Verifiable Rewards）**: RLによる推論強化の潮流だが、訓練安定性の問題は未解決だった。本論文はその課題に対応
- **RAGEN-1**: 先行研究。Agentic RLの全体的なフレームワークを提案。RAGEN-2はその中で「推論崩壊」にフォーカス
- **DeepSeek-R1（arXiv:2501.12948）**: 純粋なRLで推論能力を引き出す手法。訓練中の安定性モニタリングに本論文の知見が応用できる
- **DAPO（arXiv:2503.14476）**: オープンソースRLシステム。SNR-Aware Filteringとの統合が期待される

## まとめ

RAGEN-2は、Agentic RLにおける「見えない崩壊」を明らかにし、診断と解決の両面で実用的な枠組みを提供した。

**覚えておくべきポイント**:

1. **エントロピー ≠ 推論品質**。入力間の区別可能性（MI）も必ず確認する
2. **報酬分散が低いプロンプトは「学習価値が低い」**。フィルタリングで訓練品質を向上できる
3. **SNRの視点はAgentic RL全般に適用可能**。特定のアルゴリズムに依存しない

エントロピーだけを見て「訓練は安定している」と判断しているなら、今すぐMIもチェックした方がいい。その違いが、モデルの真の性能を左右する。
