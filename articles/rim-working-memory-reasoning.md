# RiM: 作業記憶でLLMの潜在推論を実現する

> **Reasoning in Memory** — LSTMの生みの親Sepp Hochreiterらが提案する、人間の作業記憶に着想を得た全く新しい潜在推論フレームワーク。固定メモリブロックで中間推論を「内省」し、CoTレベルの精度をdirect-answerの速度で達成する。

## TL;DR

- CoTは推論を自然言語の生成に結合しており、計算予算の一部が「文法の生成」に浪費されている
- RiMは固定の特別トークン列（メモリブロック）をLLMの作業記憶として活用し、中間推論を潜在空間で行う
- **単一のforward passで処理可能** — Coconut比で7倍高速、CoT比で27倍高速
- GPT-2 / LLaMA-1B / LLaMA-3Bで一貫してCoconutを上回り（GSM8K +2.5〜7.5pp）
- 2段階カリキュラム（段階的教師信号→回答の逐次精緻化）が鍵

## 問題背景：推論は言語に縛られすぎている

LLMの推論能力の向上は、テスト時計算（test-time compute）のスケーリングに大きく依存している。Chain-of-Thought（CoT）推論[^1]に始まり、Process Reward Models[^2]、DeepSeek-R1[^3]に至るまで、多くのアプローチが「最終回答の前に中間トークンを生成する」というパラダイムに従っている。

だが、この設計には構造的な問題がある：

1. **推論と言語生成の結合**：中間計算がオート回帰的なテキスト生成に直結し、「思考の外化」をモデルに強いる
2. **計算予算の浪費**：言語はコミュニケーション用に最適化されており、文法や流暢さの生成に計算リソースが割かれる
3. **逐次的ボトルネック**：連続表現（Coconut[^4]など）を使っても、各ステップの生成が次のステップの条件付けに必要なため、本質的に逐次的

人間の認知心理学から見れば、これは奇妙な制約だ。Vygotsky[^5]の発達心理学が示すように、人間は初期こそ「声に出して考える」が、最終的には内的処理へと移行する。Baddeley[^6]が提唱する作業記憶（working memory）は、情報を外部化することなく内部で保持・操作する能力だ。

**核心的な問い**：LLMに「声に出して考えさせる」のではなく、作業記憶で推論させることはできるのか？

![推論パラダイムの比較](/images/rim-working-memory-reasoning/fig1_paradigm_comparison.png)
*図1: CoT、Coconut、RiMの推論パラダイム比較。RiMは固定メモリブロックを単一forward passで処理するため、逐次生成のボトルネックを回避する。*

## 手法详解：RiM（Reasoning in Memory）

### メモリブロックの設計

RiMの核心は**メモリブロック（memory block）**という概念にある。各メモリブロックは、固定長の特別トークン列として定義される：

```
m_k = [<b>, <m>, ..., <m>, </b>]  （M個の<m>トークン）
```

- `<b>` / `</b>`：ブロック境界を示すデリミタ
- `<m>`：潜在作業空間を形成する特別トークン（実験ではM=2）
- K個のメモリブロックが質問に付加され、単一のforward passで処理される

**重要な設計選択**：既存語彙のトークン埋め込みは凍結し、特別トークンの埋め込みのみを学習する。これにより、事前学習済み表現の汚染を防ぎ、改善がメモリブロックの活用から生じることを保証する。

### 2段階カリキュラム

メモリブロックには事前定義された計算的役割がないため、有用な潜在計算を引き出すには慎重に構造化された教師信号が必要になる。RiMは2段階のカリキュラムでこれを実現する。

![2段階カリキュラムの構造](/images/rim-working-memory-reasoning/fig2_two_stage_curriculum.png)
*図2: Stage 1では各メモリブロックの後に次の推論ステップの予測を監督し、Stage 2では全ブロックで最終回答の精緻化を行う。*

#### Stage 1: 推論ステップ監督

各メモリブロックの直後に、その時点までの潜在空間から**次の推論ステップ**を予測させる。損失関数は：

$$\mathcal{L}_{S1}(\mathbf{w}) = -\sum_{t=1}^{T} \lambda_t(s) \log p_\mathbf{w}(r_{t+1} \mid \mathbf{x}, \mathbf{m}_{\leq t})$$

ここで $\lambda_t(s) \in [0, 1]$ は訓練ステップ $s$ における監督の強度を制御する。線形相対アニーリングを用い、早期のステップから順に監督を段階的に緩和する。

**カスタムアテンションマスク**が鍵を握る。推論ステップの読み出し（readout）は、質問とこれまでのメモリブロックのみにアテンションを向け、他の推論ステップにはアクセスできない。これによりモデルはメモリブロックを経由しないと正解を得られず、潜在空間に中間計算をエンコードすることを**強制**される。

#### Stage 2: 最終回答の精緻化

ステップレベルの監督を廃止し、固定数Kのメモリブロックで全てのサンプルを処理。各ブロック後に最終回答を予測させる：

$$\mathcal{L}_{S2}(\mathbf{w}) = -\sum_{k=1}^{K} \alpha_k \log p_\mathbf{w}(\mathbf{y} \mid \mathbf{x}, \mathbf{m}_{\leq k})$$

後のブロックほど大きい重み $\alpha_k$ を割り当て、後段の読み出しが最も強い回答を生成するよう誘導する。

### なぜCoconutのカリキュラムでは不十分か

Coconutは段階的に推論ステップを連続表現に置き換えていくが、RiMの事前実験では以下の問題が判明した：

1. **バイパス問題**：初期段階ではターゲット推論ステップが前の推論ステップにアテンションでき、監督信号が潜在空間をバイパスする
2. **打ち切り問題**：最大ステージ数で置き換えられない推論ステップが残り、監督が不均等になる

RiMは**全ステップを一斉にメモリブロックに置き換え**、カスタムマスクで全ステップに密な教師信号を供給することで、これらの問題を回避する。

## 実験結果

### メイン結果：精度とレイテンシの最適なトレードオフ

![精度とレイテンシのトレードオフ](/images/rim-working-memory-reasoning/fig3_accuracy_latency.png)
*図3: 精度 vs. レイテンシ。RiMはCoconutレベルの精度をdirect-answerの速度で達成する。Coconutは約7倍、SFT w/ CoTは約27倍遅い。*

16分割交差検証プロトコルで厳密に評価したメイン結果：

| モデル | 手法 | GSM8K (%) | GSM-Hard (%) | TTFT (ms) |
|--------|------|-----------|--------------|-----------|
| GPT-2 | SFT w/o CoT | 15.4 | 3.5 | 7.6 |
| GPT-2 | Coconut w/ Stage 0 | 31.1 | 7.1 | 53.4 |
| GPT-2 | **RiM (Final)** | **33.6** (+2.5pp) | **7.8** (+0.7pp) | **7.6** |
| LLaMA-1B | SFT w/o CoT | 23.9 | 5.3 | 16.1 |
| LLaMA-1B | Coconut w/ Stage 0 | 36.9 | 8.5 | 108.3 |
| LLaMA-1B | **RiM (Final)** | **42.1** (+5.2pp) | **10.5** (+2.0pp) | **16.1** |
| LLaMA-3B | SFT w/o CoT | 36.2 | 8.5 | 27.9 |
| LLaMA-3B | Coconut w/ Stage 0 | 41.3 | 10.2 | 188.8 |
| LLaMA-3B | **RiM (Final)** | **48.8** (+7.5pp) | **12.0** (+1.8pp) | **27.9** |

**注目すべき点**：
- RiMのTTFTはSFT w/o CoTと**完全に同一**。メモリブロックは固定入力として処理されるため、追加の推論オーバーヘッドがほぼゼロ
- Any-block読み出し（Pass@8）ではさらに劇的な差がつく。LLaMA-1BでCoconut 51.1% vs RiM **76.8%**（+25.7pp）
- DARTの公式結果を全ての設定で上回る。しかもDARTは2.5〜10倍多い訓練計算を要する

![メイン結果とメモリ予算ロバスト性](/images/rim-working-memory-reasoning/fig4_results_robustness.png)
*図4: 左：3モデルでのGSM8K精度比較。右：メモリブロック数Kを変化させた際の精度。Stage 2ではK=4〜16で精度が安定し、実用的なデプロイ性を示す。*

### メモリブロックは本当に「計算」しているのか？

PCAによる表現分析がこの疑問に答える。LLaMA-1BでStage 1（6 epoch）→Stage 2（2 epoch）の訓練中、各チェックポイントでメモリブロックの表現を取得し、共有PCA空間に射影した。

**訓練前**：メモリブロックの表現はほぼ退化（collapsed）しており、トークンIDと位置で決まる。

**訓練後**：
- ブロック固有の構造が明確に現れる（Block 1 ≠ Block 8）
- サンプル依存的なクラスタが形成される（正解/不正解で異なる方向）
- 最も遠い2サンプルの余弦類似度が**Layer 12で0.09、Layer 16で0.01**にまで低下

つまりメモリブロックは「固定されたプレースホルダー」ではなく、ブロック位置と入力問題に応じて構造化される**真の潜在作業空間**として機能している。

### メモリ予算へのロバスト性

Stage 2完了後のモデルでメモリブロック数Kとブロック内トークン数Mを変化させた実験では、K=4〜16で精度が約43%で安定。これはStage 2がStage 1で獲得された接地されたメモリブロックを、固定長の潜在計算列に変換することを示している。

### 線形プローブによる読み出し選択

メモリブロックの表現に基づく軽量線形プローブで、読み出しの正誤を予測可能。AUROC 84〜85%で、条件付き正解率は**90%**に達する。Final-blockとAny-blockのギャップはシンプルなプローブで大幅に埋められる。

## 考察

### 「推論の内省化」というパラダイムシフト

RiMの最も重要な貢献は、概念的なパラダイムシフトにある。これまでの潜在推論（Coconutなど）は「連続空間でのCoT」という枠内に留まっていたが、RiMは根本的に異なる設計原理に立つ：

- **CoT / Coconut**：推論ステップを1つずつ生成/外部化し、次のステップがそれに条件付けられる
- **RiM**：推論は固定メモリブロック内で「内省的」に行われ、外部化されない

これは発達心理学の「内言（inner speech）」から「内的処理」への移行[^5]を、LLMの訓練で実現したと言える。

### 作業記憶 vs. 逐次生成のトレードオフ

RiMの精度はCoTに及ばない部分もあるが、この比較自体が公平ではない。CoTは数百トークンの中間テキストを生成し、その分だけ推論に専念できる。RiMは**同じレイテンシでdirect-answer SFTを大幅に上回り**、CoTに近づく。

今後の方向性として、Stage 2にRLを導入することで、最終回答の報酬から潜在空間をさらに改善できる可能性が示唆されている。

### 訓練のシンプルさ

RiMの2段階カリキュラムは、いずれの段階も標準のnext-token prediction損失のみを用いる。CoconutやDARTのような多経路・多目的の学習フレームワークを必要とせず、最適化がシンプルで安定している。DARTが2つの経路で3つの損失を同時最適化するのに対し、RiMは単一forward passで学習できる。

## 関連研究との位置づけ

| 研究 | 推論形態 | 逐次性 | RiMとの関係 |
|------|---------|--------|------------|
| CoT[^1] | 離散テキスト | 逐次的生成 | ベースライン |
| Coconut[^4] | 連続表現 | 逐次生成 | 最も近い潜在推論手法、7倍遅い |
| DART[^7] | サイレント思考 | 逐次生成 | 2経路必要、2.5〜10倍高コスト |
| HRM / TRM | 再帰的精緻 | 縦方向の反復 | RiMは横方向の精緻化 |
| Abstract-CoT[^8] | 抽象トークン | 離散的だが逐次 | 近い方向性だがRiMは固定 |
| Pauser Token[^9] | 一時停止 | 付加的 | 補助的、RiMは本格的 |

RiMの独自性は、潜在推論の逐次ボトルネックを**完全に解消**した点にある。これが「固定メモリブロックの活用」というシンプルだが強力な設計選択で実現されている。

## まとめ

RiMは、人間の作業記憶に着想を得た潜在推論フレームワークとして、以下を実証した：

1. **固定メモリブロック**がLLMに真の潜在作業空間として機能する（PCA分析で裏付け）
2. **2段階カリキュラム**がこの空間を構造化・利用可能にする
3. CoT比**27倍高速**でありながら、Coconutを一貫して上回る精度を達成
4. メモリ予算に対して**ロバスト**で、実用的なデプロイ性が高い

この研究の最も深い示唆は、LLMの推論は言語の生成に縛られる必要がないということだ。計算とコミュニケーションを切り離すことで、我々は効率的でスケーラブルな推論の道を開くことができる。
