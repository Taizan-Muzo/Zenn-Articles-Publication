# DeepSeek-R1とGRPO：強化学習による推論能力の創発

## 概要

2025年1月、DeepSeekはR1モデルを発表し、大規模言語モデル（LLM）の推論能力に関する従来の常識を覆した。従来の「教師ありファインチューニング（SFT）→RLHF」という三段階パラダイムを打破し、純粋な強化学習（RL）のみで推論能力を創発させることに成功した。本稿では、この革命の核心であるGRPO（Group Relative Policy Optimization）アルゴリズムと、その技術的意義を解説する。

---

## 従来パラダイムの限界

### 標準的なLLM訓練フロー

従来の高品質LLM構築は三ステップで行われていた：

1. **言語モデリング段階**: 大規模ウェブデータで次トークン予測を学習。基盤モデル（Base Model）を生成
2. **教師ありファインチューニング（SFT）**: 指示データでモデルを微調整し、質問応答能力を付与
3. **偏好最適化段階**: RLHFやDPOで人間の偏好に合わせて最適化

### SFTの根本的問題

SFTはモデルに「人間の思考パターンを模倣」させるが、これには本質的な限界がある：

- 高品質な思考過程データの収集コストが極めて高い
- 人間の思考パターンが最適解とは限らない
- モデルは模倣に留まり、独自の推論戦略を獲得できない

---

## DeepSeek-R1の革命的アプローチ

### DeepSeek-R1-Zero：概念実証

DeepSeekはまずR1-Zeroモデルを開発した。これは完全にSFTを省略し、基盤モデル（DeepSeek-V3-Base）に直接強化学習を適用したものだ。

**報酬設計のシンプルさ**:
- **正確性報酬**: 数学問題なら最終回答の正誤を判定
- **形式報酬**: 思考過程を`<think>...</think>`タグで囲むなどの形式要件

この極めてシンプルな報酬設計のみで、モデルは自発的に「思考トークン」を生成し始めた。中間的な推論ステップを自己言語化し、自己検証や自己訂正を行う能力が、純粋にRLの繰り返しから創発したのだ。

### DeepSeek-R1：完全版の訓練パイプライン

R1-Zeroは証明に成功したが、可読性や言語混合の問題が残っていた。R1はこれを解決するため、以下のパイプラインを採用した：

1. **冷間起動（Cold Start）**: 数千件の高品質CoTデータで初期SFT
2. **第一RL段階**: 推論中心のタスクでRL訓練
3. **拒否サンプリングとSFT**: RLチェックポイントから高品質サンプルを生成し、追加SFT
4. **第二RL段階**: 全タスク範囲でRL訓練

## GRPO：高価な「批評家」を捨てる

### PPOの問題点

従来のRLアルゴリズムであるPPO（Proximal Policy Optimization）は、以下の4つのモデルを必要とする：

1. Policy Model（戦略モデル）
2. Reference Model（参照モデル）
3. Reward Model（報酬モデル）
4. **Value Model（価値モデル/批評家）**

Value ModelはPolicy Modelと同等のサイズを持ち、訓練中にメモリを倍増させる。さらに、Value Model自体の訓練（長い生成シーケンスの期待報酬を予測）は難しく、収束しにくい。

### GRPOの核心アイデア

GRPO（Group Relative Policy Optimization）はValue Modelを完全に廃止する。代わりに「グループ内相対評価」を採用する：

**具体的手順**:
1. 同一問題に対し、Policy Modelが複数の回答を生成（例：64個）
2. Reward Modelで各回答にスコアリング
3. グループ内の平均スコアをベースラインとする
4. 各回答の相対的な優劣に基づいてPolicy Modelを更新

**数式化**:

```
A_i = (r_i - mean(r)) / std(r)
```

A_iはi番目の回答の利得、r_iはその報酬、mean(r)とstd(r)はグループ内の平均と標準偏差。

この「群衆の知恵」アプローチにより、高価な「批評家」モデルなしで安定したRL訓練が可能になる。

### GRPOの損失関数

GRPOはPPOと異なり、KLダイバージェンスを報酬から分離し、損失関数に直接追加する：

```
L_GRPO = E[ (π_θ/π_old) * A ] - β * KL(π_θ || π_ref)
```

この分離により、利得計算が単純化され、訓練の安定性が向上する。

## RLVR：検証可能報酬による強化学習

DeepSeek-R1の成功は、RLVR（Reinforcement Learning with Verifiable Rewards）という枠組みを確立した。これは以下の特徴を持つ：

### 結果指向の報酬

従来のPRM（Process Reward Model）は「ステップごとの報酬」に依存し、人間による細粒度の注釈が必要だった。RLVRは「最終結果が正しいか」だけを見る。数学問題なら答えが35かどうか。途中の経路が多少遠回りでも、大規模訓練の中でモデルは自然に最短・最論理的な経路を発見する。

### KLダイバージェンスの再考

従来はモデルが「暴走」しないようKLダイバージェンスを「安全ロープ」として使用していた。しかし2025年のOlmo 3やDeepSeek-V3.2の実験は、推論タスクではこの「安全ロープ」がかえって足かせになることを示した：

- **No KL Loss**: 数学推論では、モデルは高度な探索と試行錯誤が必要。KL制約は新しい解法の発見を妨げる
- **Token-level Loss**: 文全体の損失ではなく、トークン単位で損失を計算する細粒度アプローチ

## 蒸留：小モデルへの知識転移

DeepSeek-R1の知見は、大規模モデルから小規模モデルへの蒸留にも応用された。R1の出力を用いてQwenやLlamaシリーズをファインチューニングすることで、驚異的な結果が得られた：

| モデル | MATH-500 | AIME 2024 |
|--------|----------|-----------|
| GPT-4o | - | 約40% |
| DeepSeek-R1-Distill-Qwen-1.5B | 約60% | 約28% |
| DeepSeek-R1-Distill-Qwen-7B | 約80% | 約55% |
| DeepSeek-R1-Distill-Qwen-32B | 約90% | 約72% |

1.5BパラメータのモデルがGPT-4oを上回る性能を示すケースもある。これは「小モデル+高品質データ」が「大モデル+低品質データ」を凌駕しうることを示す。

## 技術的意義と今後の展望

### 推論訓練の民主化

GRPOとRLVRの組み合わせにより、推論モデルの訓練コストが劇的に低下した。Value Modelが不要になったことで、一般的な研究機関でも大規模RL実験が可能になった。

### 自己改善のループ

DeepSeek-R1は、モデルが人間の教師なしに「自分で考え、自分で改善する」ループに入ることを示した。これはAGI（汎用人工知能）への重要な一歩と見なされている。

### 今後の研究方向

1. **Verifier & meta-Verifier**: 推論過程の検証を強化する新たな枠組み
2. **長期推論の最適化**: より長い思考チェーンを効率的に訓練する手法
3. **マルチモーダル推論**: 視覚・音声を含む推論への拡張
4. **継続学習**: 推論能力の継続的な向上メカニズム

---

## 結論

DeepSeek-R1とGRPOは、2025年を「推論元年」に変えた。教師あり学習への依存を減らし、強化学習による自己創発的な推論能力の獲得が現実になった。このパラダイムシフトは、今後のLLM開発において「スケールだけでなく、思考の質」が重要になることを示唆している。
