# はじめに

LLMの推論能力が飛躍的に向上する一方で、「**どれだけ思考を続ければよいか**」という根本的な問いに対する体系的なアプローチは驚くほど限られている。推論チェーンの長さは推論コストとパフォーマンスの双方を直接左右するが、既存の長さ制御手法はシーケンスレベルの粗いペナルティに依存しており、token単位のきめ細かな制御ができないままだった。

今回紹介する **LenVM（Length Value Model）**（arXiv: 2604.27039）は、この問題に**強化学習の価値推定というレンズ**で切り込む。UCSBとAppleの共同研究チームが提案するこのフレームワークは、生成长度を「各tokenに定数負の報酬を与えた場合の割引累積報酬」として定式化し、**無監督・密・無偏かつスケーラブル**なtokenレベルの長さ信号を実現する。2026年4月29日公開。

![fig1](/images/lenvm-length-value-model/fig1_value_framework.png)

# TL;DR

- 生成长度モデリングを**RLの価値推定問題**として定式化。各tokenに定数負報酬 $r=-1$ を付与し、割引累積報酬で残り生成horizonを予測
- sigmoid変換で値を $(-1, 0)$ の有界区間にマッピングし、ベルマン再帰との整合性を保証
- **LIFEBench**でスコア30.9→**64.8**（7Bモデル）。先進クローズドソースモデルのprompt制御を凌駕
- GSM8Kで200 token予算下に**63%維持**（硬截断6%に対して**10倍以上**）
- 32Bモデルでの平均相対誤差9.8%を達成。スケーラビリティが実証されている

# なぜtokenレベルの長さモデリングが必要なのか

推論モデルにとって生成长度の制御は極めて重要だ。思考が短すぎれば複雑な問題に答えられず、長すぎれば計算コストが無駄に増大する。しかし、現在のアプローチには構造的な限界がある。

## 既存手法の限界

**シーケンスレベルの罰**: 長さに比例した負の項をlossに加える手法（DLERなど）は、すべてのtokenに一律の圧力をかけるだけで、どのtokenが長さの主導要因かを区別できない。

**プロンプトベース**: 「100語以内で答えて」のような指示に依存する方法は、モデルの遵守度に大きく左右され、特に低性能なモデルでは機能しない。

**硬截断（Hard Truncation）**: 生成を一定のtoken数で強制終了する方法は、最も単純だが破壊的だ。GSM8Kで200 tokenのハードカットをかけると、精度はたったの6%まで低下する。

LenVMが着目するのは、**「どのtokenが生成の延長を促し、どのtokenが終了を導くのか」**という、生成動態のtokenレベル構造だ。

# LenVMのコア定式化

## 強化学習としての長さモデリング

LenVMの核心は極めてシンプルだ。生成を**有限ホライズンのMDP**とみなす：

- **状態 $s_t$**: デコーディング時点 $t$ におけるコンテキスト
- **行動 $a_t$**: 次のtokenの選択
- **報酬 $r_t$**: $r_t = -1$（すべてのtoken生成に対して定数）
- **価値 $V(s_t)$**: 残り生成horizonの単調プロキシ

各ステップで1token生成すると「-1のコスト」を支払うと考える。すると、残りの生成长度は次の割引累積報酬になる：

$$G_t = \sum_{k=0}^{L-t} \gamma^k \cdot (-1) = -\frac{1 - \gamma^{L-t}}{1 - \gamma}$$

これをsigmoid変換で $(-1, 0)$ に有界化する：

$$V(s_t) = -\sigma(\alpha \cdot G_t + \beta)$$

## 監督信号の4つの特性

この定式化がもたらす信号は4つの重要な性質を備えている：

| 特性 | 従来手法 | LenVM |
|------|---------|-------|
| アノテーション | 要（長さラベル） | **不要**（自己完結） |
| 密度 | 疎（シーケンス単位） | **密**（全token位置） |
| バイアス | 報酬モデル由来のノイズ | **無偏**（決定論的計算） |
| スケーラビリティ | データボトルネック | **無限のデータ**（生成自体が教師） |

とりわけ「無監督」の点が重要だ。既存の推論データに対して完了済みの応答があれば、それから自動的に価値targetを計算できる。追加のアノテーションコストは**ゼロ**。

# 推論時の統合：長さガイダンス

![fig2](/images/lenvm-length-value-model/fig2_lifebench.png)

LenVMは推論時に**指数傾斜（Exponential Tilting）**を使って、次token分布を長さ方向に誘導する：

$$p_\theta(a_t | s_t, \lambda) \propto p_\theta(a_t | s_t) \cdot \exp(\lambda \cdot V(s_t'))$$

ここで $V(s_t')$ は次のステップのLenVM価値、$\lambda$ は制御パラメータだ。$\lambda < 0$ で短い生成を、$\lambda > 0$ で長い生成を促すことができる。

これが「硬截断」と決定的に違うのは、**生成の道筋を変える**点だ。硬截断は完了した思考を暴力的に中断するが、LenVMはモデルを短い成功軌跡に**誘導**する。ベースモデルがすでに「短くても正解できる」軌跡を持っていれば、LenVMはそれを効果的に引き出せる。

# 実験結果

## LIFEBench：正確な長さマッチング

LIFEBenchは「正確にN token出力せよ」「最大N token」「最低N token」の3つの制約設定で長さ制御能力を評価するベンチマークだ。

結果は圧巻だった。Qwen2.5-7Bベースラインのスコア30.9が、LenVMのガイダンスによって**64.8**まで引き上げられた。これはpromptベースで制御するGPT-4o（38.2）、Claude-3.5（42.5）、Gemini-2.0（45.1）を**すべて上回る**。

7Bオープンソースモデルが、プロンプト制御するクローズドソースの先進モデルを**20点近く上回る**という結果は、tokenレベルの価値信号がどれほど強力かを示している。

## GSM8K：性能-効率の連続制御

![fig1](/images/lenvm-length-value-model/fig1_value_framework.png)

ここが実用上最も重要な結果だ。200 tokenの予算制約下で：

- **LenVMガイダンス**: 63%の精度を維持
- **硬截断**: たった6%

10倍以上の差だ。この結果は、**ベースモデルが「短くても正解にたどり着ける」潜在的な推論軌跡を多数持っている**こと、そしてLenVMがそれを特定して引き出せることを示している。

予算を600 tokenまで緩めると、LenVMガイダンスは89%に到達し、制約なしの92%にほぼ迫る。この連続的な制御こそが、実運用におけるコスト管理に不可欠だ。

## 生成长度予測

32Bモデルの数学タスクで、prompt境界から総生成长度を予測した結果、平均相対誤差（MRE）は**9.8%**。これは推論前のメモリ割り当てやバッチ処理の最適化に直接応用できる。モデルスケールとともにMREは単調減少し、0.5Bの28.5%から32Bの9.8%まで改善する——スケーラビリティの強い証拠だ。

# 消融実験：なぜこの定式化が機能するのか

![fig3](/images/lenvm-length-value-model/fig3_ablation.png)

## 目標関数の設計選択

4つの目標パラメータ化手法を比較した：

1. **Raw Length Regression**: 生のtoken長を直接回帰 → 評価loss 0.42
2. **Normalized Length**: 長さを正規化 → 0.31
3. **Discounted Return**: 割引累積報酬 → 0.19
4. **Discounted + Sigmoid**: 割引累積報酬をsigmoid変換 → **0.13**（最良）

生の長さ回帰が最悪なのは、長さ分布が強く右に歪んでいるためだ。一部の超長出力がlossを支配してしまう。一方、割引累積報酬 + sigmoid変換は、自己回帰的なベルマン再帰と整合し、各ステップで一貫した予測を可能にする。

## 割引因子 $\gamma$ のトレードオフ

$\gamma$ の選択には微妙なトレードオフがある：

- **高い $\gamma$（0.97-0.99）**: 長いhorizonを圧縮し、早期段階での予測を容易にするが、終了付近の分解能が落ちる
- **低い $\gamma$（0.80-0.85）**: 終了付近の区別が明確になるが、初期段階の予測精度が低下

実験では **$\gamma = 0.95$** が最適バランスを提供した。

## バッチ構築戦略

異なるpromptの完了結果をバッチ内でシャッフルする戦略が、promptごとにグループ化する戦略を一貫して上回った。これは過学習の抑制と汎化能力の向上を示唆している。

# スケーラビリティと解釈可能性

![fig4](/images/lenvm-length-value-model/fig4_scalability_interpret.png)

## スケーラビリティの実証

LenVMの評価lossは3つの次元で**単調減少**する：

- **モデルサイズ**: 0.5B → 32B でlossが大幅減少
- **訓練prompt数**: より多くのpromptでlossが低下
- **各promptのサンプル数**: サンプル増加でlossが改善

これは「生成自体が教師データになる」スケーラビリティの強みだ。生成が増えれば増えるほど、LenVMの価値予測は精緻になる。

## Tokenレベルの解釈可能性

LenVMのTD残差（Temporal Difference Residual）を分析すると、特定のtokenが生成動態の転換点として浮かび上がる：

- **正のTD残差**（生成延長）: 「ah」「but」「now」「wait」「let」「think」「however」——モデルが推論を修正・拡張するタイミング
- **負のTD残差**（終了シグナル）: 「therefore」「clearly」「done」「result」「✓」「sum」——推論の閉じを示唆

この分析は、LenVMが単なる長さ予測器ではなく、**推論プロセスの構造的理解を提供する**解釈可能性ツールとして機能することを示している。

# RLとの親和性

LenVMの価値定式化は、PPOスタイルのRLと**本質的に互換性がある**：

- **価値ベースライン**: LenVMの価値を長さ特有の報酬のベースラインとして利用可能
- **報酬シェーピング**: tokenレベルの势関数としてRLの報酬設計を補助
- **密な信用割り当て**: ポリシー不変なtokenレベル信号で信用割り当てを改善
- **品質と効率の分離**: タスク品質の報酬と生成効率の報酬を独立に最適化

特に最後の点が重要だ。現在のRLVRでは、推論の品質向上と計算効率化が暗黙に結びついているが、LenVMはこの2つを明示的に分離する基盤を提供する。

# 関連研究との位置づけ

LenVMは、2025年以降に活発化している**推論長さ制御**の研究ラインに位置づけられる：

- **L1**（2025）: 長さ制御にRLを直接適用。LenVMはRLの前段に位置し、価値予測を提供する
- **Just Enough Thinking**（2025）: 適応的長さペナルティで効率化。シーケンスレベルで、LenVMはtokenレベル
- **Can LLMs Track Their Output Length?**（2026）: モデルの長さ認識能力を分析。LenVMはそれを**能動的に改善**する
- **Predicting LLM Output Length via Entropy**（2026）: エントロピーベースの予測。LenVMはRLベースでより密な信号

LenVMの独自性は、**長さを「価値」として定式化する**点にある。これにより、RLの理論的ツールキットがそのまま活用できる。

# 考察：実用への展望と限界

## 強み

LenVMの最大の貢献は、長さ制御という実用上の問題に**理論的に洗練された定式化**をもたらしたことだ。「各tokenは-1のコストを支払う」という極めて直感的なMDP構造が、無監督・密・無偏という望ましい性質をすべて同時に満たすのは美しい。

GSM8Kで200 token予算下に63%を維持した結果は実用上のインパクトが大きい。推論APIのコスト削減において、質を落とさずにトークン消費を制御できる能力は直接的な価値を持つ。

## 課題

一方で、いくつかの限界も指摘できる：

- **タスク依存性**: 数学推論（GSM8K）でのみ検証されており、オープンエンド生成やコード生成での効果は未検証
- **価値推定の精度限界**: 32BモデルでもMRE 9.8%であり、厳密な長さ制御（LIFEBenchのExact）では64.8に留まる
- **$\lambda$ のチューニング**: 最適な制御パラメータ $\lambda$ はタスクごとに調整が必要な可能性がある

## 今後の方向性

LenVMの価値信号をRLVRに統合する方向性が最も魅力的だ。推論長さを制御可能にした上で、品質の最大化をRLで行う二段階最適化は、現実のデプロイメントにおいて非常に合理的だ。

# まとめ

LenVMは、LLMの生成長さモデリングをRLの価値推定として再定式化する。tokenレベルの密な価値信号が、粗いシーケンスレベル手法の限界を突破し、7Bモデルで先進クローズドソースモデルを凌駕する長さ制御を実現する。推論コストの最適化が喫緊の課題となる中、LenVMのアプローチは実運用に直結する強力な基盤技術だと言える。
